# Agentic Reinforced Policy Optimization

## 基本信息

- **标题**: Agentic Reinforced Policy Optimization
- **作者**: Guanting Dong, Hangyu Mao, Kai Ma, Licheng Bao, Yifei Chen, Zhongyuan Wang, Zhongxia Chen, Jiazhen Du, Huiyang Wang, Fuzheng Zhang, Guorui Zhou, Yutao Zhu, Ji-Rong Wen, Zhicheng Dou

- **arXiv ID**: 2507.19849
- **arXiv 链接**: <https://arxiv.org/abs/2507.19849>
- **领域/关键词**: ARPO, Agentic RL, Multi-turn RL, Tool Use, LLM Agents, Entropy-based Rollout, Adaptive Sampling

## 一句话总结

> ARPO 是一种面向多轮工具交互场景的智能体强化学习算法，通过基于熵的自适应展开机制和优势归因估计，在工具调用后的高不确定步骤进行分支采样，仅用一半的工具调用预算即超越传统轨迹级 RL 方法。

## 论文精读

### 背景与前置知识

**RLVR (Reinforcement Learning with Verifiable Rewards)** 是当前提升大语言模型推理能力的主流范式。它通过可验证的奖励信号（如数学题的正确与否）来训练 LLM，已在单轮推理任务（如数学、代码生成）中取得了显著成效。代表作包括 OpenAI o1、DeepSeek-R1、QwQ 等。

**多轮工具使用 (Multi-turn Tool Use)** 是指 LLM 在推理过程中与外部工具（搜索引擎、代码解释器、浏览器等）进行多轮交互的能力。不同于单轮推理，多轮工具使用要求模型不仅能进行长链推理，还需在推理中动态调用工具、解析反馈、做出决策。

**LLM Agents** 是指以 LLM 为核心决策引擎的智能体系统。这类智能体需要具备长程规划、自适应决策和与外部环境动态交互的能力。Agentic RL 正是为此设计的训练范式，将 LLM 训练从静态任务求解扩展到动态的智能体-环境交互推理。

当前 Agentic RL 的主流方法（如 GRPO、DAPO）采用**轨迹级算法**：独立采样完整的工具使用轨迹，基于最终输出提供奖励信号。然而，这些方法忽略了工具调用反馈后模型推理的不确定性变化，无法有效探索细粒度的逐步工具使用行为。

### 核心思想详解

ARPO 的核心思想源于一个关键发现：**LLM 在接收到工具调用反馈后，其生成 token 的熵值会显著升高**，表明模型在这些步骤面临高度不确定性。

这一现象的直觉解释是：
1. 外部工具反馈与模型内部推理之间存在**分布偏移**，导致模型需要重新适应新的信息分布
2. 搜索引擎返回的丰富文本内容比 Python 执行的确定性数值带来更大的熵值波动
3. 每次工具调用后的前 10-50 个 token 的熵值上升最为明显

基于此，ARPO 提出了两个核心机制：

**Entropy-based Adaptive Rollout（基于熵的自适应展开）**：不是对完整轨迹进行均等采样，而是在工具调用后熵值显著变化的步骤进行分支采样。这样可以在高不确定性的推理节点投入更多探索资源，发现更优的逐步工具使用策略。

**Advantage Attribution Estimation（优势归因估计）**：由于自适应展开会产生包含共享前缀和分支路径的轨迹，需要一种区分共享 token 和分支 token 的优势估计策略，使模型能有效内化逐步工具使用行为中的优势差异。

### 方法逐步拆解

ARPO 算法的执行流程可分为以下步骤：

**步骤一：Rollout 初始化**
- 给定全局展开大小 M，LLM 首先通过轨迹级采样生成 N 条轨迹
- 剩余 M - N 条轨迹预算保留给部分采样
- 计算每条轨迹前 k 个 token 的熵值，形成初始熵矩阵 H_initial

**步骤二：熵变化监控**
- 模型开始带有工具的智能体推理
- 每次工具调用后，模型生成 k 个额外 token
- 计算步骤级熵矩阵 H_t，并通过归一化公式计算相对于初始状态的熵变化量 Delta_H_t

**步骤三：基于熵的自适应分支**
- 定义部分采样概率 P_t = alpha + beta * Delta_H_t
- 当 P_t 超过预设阈值 tau 时，从当前节点分支 Z 条部分推理路径
- 否则继续沿当前轨迹推理
- 这种机制使模型能自适应地将探索资源分配给高不确定性的推理空间

**步骤四：终止条件**
- 迭代直到满足以下条件之一：(1) 分支路径总数达到预算 M - N；(2) 所有路径在达到预算前已终止
- 若所有路径提前终止，则补充额外的轨迹级采样以满足预算

**步骤五：优势归因估计**
- **硬优势估计**：共享 token 获得所有包含该共享段的轨迹的平均优势值；独立 token 获得各自的归一化奖励优势
- **软优势估计**：利用 GRPO 的重要性采样比率自动区分共享和独立 token，共享前缀的 token 具有相同的重要性权重
- 实验表明软优势估计在训练中更稳定、奖励更高，因此 ARPO 默认使用软设置

### 关键公式/算法解读

**1. 智能体 RL 训练目标（公式1）**

训练目标是在最大化期望奖励的同时约束策略偏移：max E[r_phi(x,y)] - beta * D_KL[pi_theta || pi_ref]

其中 T 为可用工具集合，pi_theta 为策略 LLM，pi_ref 为参考 LLM，r_phi 为奖励函数，D_KL 为 KL 散度。

**2. 智能体推理展开分解（公式2）**

P_theta(R, y | x; T) = [Agentic Reasoning 部分] * [Answer Generation 部分]

将推理过程分解为工具调用推理阶段 R（与工具交互）和最终答案生成阶段 y，这体现了多轮交互的本质。

**3. Token 熵计算（公式3）**

H_t = -sum(p_{t,j} * log p_{t,j})，其中 p_t = pi_theta(* | R_{<t}, x; T)

计算每个 token 位置上生成分布的熵值，反映模型在该步骤的不确定性程度。

**4. 归一化熵变化（公式4）**

Delta_H_t = Normalize(H_t - H_initial)

量化工具调用后熵值相对于初始状态的变化，正值表示不确定性增加。

**5. 自适应分支概率（公式5）**

P_t = alpha + beta * Delta_H_t，当 P_t > tau 时触发分支采样

alpha 为基础采样概率，beta 为熵权重系数，tau 为阈值。该公式将熵变化转化为分支决策。

**6. 分层奖励设计（公式8）**

综合正确性奖励和格式奖励，并引入多工具协作奖励 r_M：当模型正确回答问题、遵循工具调用格式且同时使用搜索和代码工具时获得额外奖励。

**7. 广义策略梯度定理（公式9，理论贡献）**

将 Transformer 的输出 token 序列划分为 K 个宏动作段，推导出适用于所有 Transformer 策略的广义策略梯度定理，为 ARPO 的部分展开机制提供了理论基础。

### 实验设计分析

**数据集覆盖**：共 13 个基准测试，涵盖三大领域：
- 数学推理：AIME2024、AIME2025、MATH500、MATH、GSM8K（5 个）
- 知识密集型推理：WebWalker、HotpotQA、2WikiMultihopQA、MuSiQue、Bamboogle（5 个）
- 深度搜索：GAIA、WebWalkerQA、HLE、xbench-DeepSearch（4 个）

**基线方法**：
- 直接推理：Qwen2.5-Instruct、Llama3.1-Instruct、Qwen3 系列，以及 GPT-4o、DeepSeek-R1-671B 等闭源大模型
- 轨迹级 RL：GRPO、DAPO、REINFORCE++
- 工作流方法：Vanilla RAG、Search-o1、WebThinker、ReAct

**训练设置**：
- 冷启动 SFT 阶段：使用 Tool-Star 的 54K 开源数据 + STILL 0.8K 数据
- RL 阶段：深度推理任务使用 10K 样本，深度搜索任务仅用 1K 样本
- 基于 VERL 框架实现，使用 8/16 块 NVIDIA H800 GPU

**关键实验结果**：
1. ARPO 在所有 10 个推理基准上平均提升约 4%，在不同骨干模型（Qwen、Llama）上一致优于 GRPO、DAPO、REINFORCE++
2. 在深度搜索任务中，Qwen3-14B + ARPO 在 GAIA 上达 43.7%，HLE 上达 10.0%，仅用 1K RL 样本即超越 GPT-4o
3. ARPO 仅需 GRPO 一半的工具调用预算即可达到更优性能
4. Pass@K 分析显示 ARPO 具有更好的采样多样性和推理效率
5. 缩放分析表明 ARPO 性能随全局展开大小增加而提升，具有可扩展性

## 问题与动机

当前 Agentic RL 面临的核心问题是：轨迹级 RL 方法（GRPO、DAPO 等）侧重于完整轨迹的比较采样，忽略了工具调用反馈后模型推理的细粒度不确定性变化。这导致：
- 探索多样性不足，无法发现高效的逐步工具使用策略
- 训练效率低下，工具调用开销大
- 难以平衡模型的内在长链推理能力和多轮工具交互能力

ARPO 的动机是通过量化 token 级熵变化，在高不确定性步骤自适应地增加探索，从而在不显著增加计算成本的前提下提升智能体的工具使用能力。

## 核心方法

ARPO 由两个核心组件构成：
1. **Entropy-based Adaptive Rollout**：监控工具调用后的熵变化，在高熵步骤进行分支采样，平衡全局和步骤级采样
2. **Advantage Attribution Estimation**：区分共享和独立 token 的优势值，帮助模型内化逐步工具使用行为的优势差异

配合分层奖励设计（正确性 + 格式 + 多工具协作），ARPO 形成完整的训练框架。

## 实验结果

- **数据集**: 13 个基准测试（5 数学 + 5 知识推理 + 3 深度搜索）
- **主要指标**: 准确率（Accuracy）、Pass@1/3/5、F1 分数、工具调用效率
- **关键结果**: 平均提升约 4%，工具调用预算仅需基线一半，Qwen3-8B+ARPO 在 GAIA 达 53.9%，超越 GPT-4o
- **对比方法**: GRPO、DAPO、REINFORCE++、Vanilla RAG、Search-o1、WebThinker、ReAct、GPT-4o、DeepSeek-R1-671B

## 局限性

1. 实验主要在 Qwen 和 Llama 系列模型上验证，对其他模型架构的泛化性有待进一步确认
2. 熵阈值、分支概率参数（alpha、beta、tau）需要针对不同任务进行调整
3. 深度搜索任务仅使用 1K 训练样本，更大规模训练下的表现未充分探索
4. 浏览器智能体能力与推理模型表现高度相关，更强的外部工具是否同样适用尚不明确
5. 论文未讨论 ARPO 在非工具使用场景（如纯文本推理）中是否也能带来增益

## 个人思考

ARPO 的工作为 Agentic RL 提供了一个非常有价值的视角：从 token 级不确定性出发设计采样策略。其基于熵的自适应展开机制简洁而有效，将探索资源集中在模型最"困惑"的推理步骤上，这与人类在面对新信息时需要更多思考的直觉一致。

优势归因估计的设计也很巧妙——通过重要性采样比率自然地区分共享和独立 token，避免了复杂的显式分段操作。软优势估计比硬优势估计更优的实验结果也印证了"隐式区分优于显式区分"的设计哲学。

值得关注的是 ARPO 的高效性：仅用一半工具调用预算和 1K RL 样本就超越了大规模模型。这对于实际部署中控制 API 调用成本具有重要意义。

一个潜在的研究方向是将熵引导的自适应采样思想扩展到其他多模态交互场景，如视觉-语言智能体、机器人控制等。此外，动态调整熵阈值（如课程学习策略）可能是进一步提升 ARPO 性能的有效途径。
