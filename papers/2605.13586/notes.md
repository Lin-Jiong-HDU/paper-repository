# HetScene: Heterogeneity-Aware Diffusion for Dense Indoor Scene Generation

## 基本信息

- **标题**: HetScene: Heterogeneity-Aware Diffusion for Dense Indoor Scene Generation
- **作者**: Zini Chen, Junming Huang, Rong Zhang, Jiamin Xu, Cheng Peng, Chi Wang, Weiwei Xu
- **发表日期**: 2026年5月13日
- **arXiv ID**: 2605.13586
- **arXiv 链接**: <https://arxiv.org/abs/2605.13586>
- **领域/关键词**: 3D scene generation, 室内场景, 异质性感知, 扩散模型, 主次物体

## 一句话总结
>
> HetScene 提出了一种异质性感知的两阶段扩散框架，通过将室内物体分解为主物体（床、沙发等大件家具）和次物体（书本、台灯等小件物品）分别生成，解决了现有方法在稠密室内场景中难以兼顾全局布局和局部细节的问题。

## 论文精读

### 背景与前置知识

**室内场景生成**：目标是自动生成合理的 3D 室内布局——确定房间中应该放置哪些家具以及它们的位置、朝向和尺寸。这是计算机图形学和具身智能（Embodied AI）的基础技术，用于为机器人训练提供模拟环境。

**主物体 vs 次物体**：这篇论文将室内物体分为两类——主物体（Primary Objects）如床、沙发、餐桌、橱柜，它们体型大、数量少、主要受房间整体结构约束（靠墙、对齐等）；次物体（Secondary Objects）如书籍、台灯、枕头、装饰品，它们体型小、数量多、强烈依赖附近的主物体（例如台灯必定在桌子或床头柜上）。

**扩散模型（Diffusion Model）**：一种生成模型，通过逐步向数据添加噪声然后学习逆向去噪过程来生成新数据。在布局生成中，扩散模型从随机噪声出发，逐步"雕琢"出合理的物体布局。

**场景图（Scene Graph）**：一种表示场景中物体之间空间关系的数据结构，节点代表物体，边代表关系（如"在左边"、"紧挨着"、"在...上面"）。

### 核心思想详解

HetScene 的核心洞察可以类比为**室内设计中的"先搭骨架，再填细节"原则**。

想象你要布置一个房间：你首先会决定大件家具的摆放位置——床放哪里，沙发靠哪面墙，餐桌在哪个区域。这些决定了房间的基本功能分区。然后，你才会考虑小物件的摆放——床头柜上放台灯，书桌上放电脑，沙发前放茶几。

然而，现有的深度学习方法（如 DiffuScene、MiDiffusion）把所有物体视为"同等重要"，在一个统一的生成过程中同时决定大件家具和小物件的布局。这就像要求设计师同时考虑所有东西——大床的位置和小台灯的位置在同一个优化过程中决定，导致相互干扰。大件家具需要全局考虑（靠墙、留出通道），小物件需要局部考虑（放在某件家具上），两者的统计规律完全不同，混在一起训练会让模型顾此失彼。

HetScene 的解决方案是**显式地将生成过程分解为两个阶段**：

1. **结构布局生成（SLG）**：只生成主物体，关注房间整体的功能组织——分区、对齐、通道等全局约束。
2. **上下文布局生成（CLG）**：以第一阶段生成的主物体布局为"锚点"，围绕它们生成次物体，关注局部支撑关系。

这种分解将原来的"所有物体联合生成"问题转化为"全局规划 + 局部填充"的组合，每个阶段只处理与自己规律匹配的子集，大大降低了问题的复杂度。

### 方法逐步拆解

#### 步骤 1：层次化场景表示

首先定义主物体集合 S^L（64 个类别）和次物体集合 S^S（37 个类别）。每个物体 o_i 用一个四元组表示：(c_i, t_i, s_i, θ_i)，分别代表类别（one-hot 编码）、位置（平移向量）、尺寸（半尺寸向量）、朝向（角度编码为 cos/sin）。主物体数量上限 N_L < 20，次物体上限 N_S < 100。使用固定大小的槽位矩阵表示，缺失位置用空类别填充。

#### 步骤 2：结构布局生成（SLG）

SLG 阶段训练一个扩散模型，条件输入包括：
- **文本描述**：用 BERT 编码自然语言描述，通过交叉注意力注入
- **房间掩码**：二值化的房间平面图，用 ResNet-18 提取特征
- **可学习场景图（LSG）**：显式编码物体间的空间关系

LSG 是本文的一个重要创新。它编码一个有向图 G = (V, E)，其中节点是物体，边是关系（方向、距离、对齐等）。每条边用源节点和目标节点的嵌入拼接后通过 MLP 编码，再通过一个轻量级 Transformer 编码器处理所有边特征，最后用掩码平均池化得到一个全局图 token，通过交叉注意力指导生成过程。这种方式让模型超越了简单的几何规则，能够自主学习隐式的空间语义。

#### 步骤 3：上下文布局生成（CLG）

CLG 阶段复用与 SLG 相同的 Transformer 骨干网络。它将 SLG 生成的主物体布局 X̂^L（固定，不加噪声）与带噪声的次物体 X^S_t 沿序列维度拼接成 N_L + N_S 长的 token 序列，通过自注意力机制让每个次物体 token 同时关注主物体和其他次物体。训练时只对次物体部分反向传播梯度。

#### 步骤 4：可学习空间-语义调制机制

这是论文中的另一个关键技术细节。在将空间嵌入（位置坐标）和语义嵌入（类别）组合成 token 时，简单的相加会导致数值较大的几何信号淹没分类信号，在稠密场景中无法收敛。解决方案是引入一个可学习的标量 γ_pos 来加权空间嵌入：

x' = x + γ_pos · e_pos

通过将 γ_pos 初始化为极小的值，确保训练初期空间信号不会从数值上压过语义属性，实现多模态特征的平衡优化。论文发现这个单标量校准是稠密场景中第二阶段的收敛前提。

### 关键公式/算法解读

**扩散模型基础**：

前向过程：q(x_t | x_0) = N(x_t; √ᾱ_t · x_0, (1 - ᾱ_t)I)

这表示将干净数据 x_0 逐步加噪，t 时刻的数据服从以 √ᾱ_t·x_0 为均值、(1-ᾱ_t)I 为方差的正态分布。

网络训练目标：L_reverse = E[||ϵ - ϵ_θ(x_t, t)||²]

网络 ϵ_θ 学习预测添加的噪声 ϵ，通过最小化预测噪声和实际噪声的均方误差。

**附加 IoU 损失**：L = L_noise + λ · L_IoU

除了预测噪声，还增加了 IoU 损失惩罚预测边界框之间的重叠，减少物体碰撞。

**空间-语义调制**：x' = x + γ_pos · e_pos

通过可学习标量 γ_pos 控制空间嵌入 e_pos 的注入强度。

### 实验设计分析

**数据集**：使用 M3DLayout 基准，它整合了 3D-FRONT（合成数据）、Matterport3D（真实扫描）和 Inf3DLayout（程序化生成）三个来源，覆盖 101 个语义类别（64 主 + 37 次），包含 11,508 个训练场景和 1,496 个验证场景。

**评估指标**：
- **FID/KID**（Fréchet/Kernel Inception Distance）：在三个正交平面（俯视图 XZ、侧视图 XY/YZ）上计算边界框投影的分布距离，衡量生成布局与真实布局的整体分布匹配程度。
- **CLIP Score**：衡量生成场景与输入文本的语义一致性。

**对比方法**：ATISS（自回归模型）、DiffuScene（扩散+UNet）、MiDiffusion（离散-连续混合扩散）。

**关键结果解读**：

1. **全测试集表现**：HetScene 在所有三个投影平面上均取得最优 FID 和 KID。俯视图（XZ 平面）FID 从 16.71（MiDiffusion）降至 14.76，提升约 11.7%。这说明 HetScene 更好地学习了与真实场景对齐的联合分布。

2. **按数据源分解**：在 3D-FRONT（布局逻辑强）和 Inf3DLayout（次物体密集）上表现最佳。在 MP3D（真实扫描噪声大）上的优势主要在侧视图，这与真实扫描数据的域偏移有关。

3. **消融实验**：逐步添加 Transformer 骨干、异质性流水线（HP）、可学习场景图（LSG），各项指标持续改善，验证了每个组件的有效性。CLIP 分数在各配置间保持稳定，说明增强物理合理性没有牺牲语义一致性。

4. **定性比较**：基线方法频繁出现物体漂浮（蓝色框标记）或结构重叠（红色框标记）问题，HetScene 则生成物理合理的稠密布局。

## 问题与动机

现有的基于深度学习的室内场景生成方法通常将所有物体视为同质实例，在统一的生成过程中处理。这种方法在稀疏简单布局中有效，但在稠密物体排列和复杂空间依赖关系中表现不佳——主物体（大件家具）受全局房间约束，次物体（小件物品）强烈依赖于附近主物体，两者的统计和几何分布完全不同。混合在一起训练导致模型难以扩展至真实世界的物体丰富场景。

## 核心方法

HetScene 提出异质性感知的两阶段扩散框架：**第一阶段（SLG）** 基于文本描述、房间掩码和可学习场景图条件，生成主物体的全局结构布局，建立房间功能骨架；**第二阶段（CLG）** 以生成的主物体布局为空间上下文和支撑锚点，通过序列拼接自注意力机制生成稠密的次物体布局。框架还包括可学习空间-语义调制机制，确保多模态特征在稠密场景中的稳定训练。可学习场景图（LSG）显式编码物体间的空间关系作为条件信号。

## 实验结果

- **数据集**: M3DLayout（含 3D-FRONT、Matterport3D、Inf3DLayout）
- **主要指标**: FID（Fréchet Inception Distance）、KID（Kernel Inception Distance）、CLIP Score
- **关键结果**: 全测试集上所有投影平面最优——XZ-FID 14.76（第二好 16.71），XY-FID 12.23，YZ-FID 12.59；消融实验验证每个组件有效性
- **对比方法**: ATISS（自回归）、DiffuScene（扩散+UNet）、MiDiffusion（混合扩散）

## 局限性

1. **对噪声数据的鲁棒性有限**：在真实扫描数据（MP3D）上，俯视图优势缩小，可能受域偏移影响。
2. **固定的主次物体划分**：主/次物体的分类是人为定义的，某些场景中物体的角色可能有变化。
3. **固定槽位上界**：主物体上限 20、次物体上限 100 的设定可能无法覆盖特别复杂的房间。
4. **依赖 M3DLayout 数据集的 3D 资产库**：场景可视化需要资产检索，受限于资产库的覆盖范围。
5. **计算开销**：两阶段生成流程需要分别运行两个扩散过程，计算时间可能比单阶段方法更长。

## 个人思考

HetScene 的核心贡献在于提出了一个直觉上非常合理的分解思路——将室内物体按功能角色分为主次两个层次分别生成。"先骨架再细节"的设计原则不仅适用于室内场景，也可能推广到其他结构化生成任务（如城市布局、文档排版）。

可学习场景图（LSG）的设计很有意思——它不是简单地将关系编码为离散标签，而是通过可学习的嵌入让模型自主发现空间语义模式。这种方法比传统的硬编码几何规则更灵活，也比完全隐式地依赖自注意力更可控。

空间-语义调制机制虽然只是一个简单的可学习标量 γ_pos，但反映了深度学习中一个常见但容易被忽视的问题：多模态特征组合时，不同模态的数值尺度差异会导致训练不稳定。这种"小细节大影响"的问题在实际工程中非常普遍。

对于进一步改进，可以考虑将主次物体的划分也变成可学习的而不是人工定义，或者引入自适应阈值来动态区分主次物体。
