# AcquisitionSynthesis: Targeted Data Generation using Acquisition Functions

## 基本信息

- **标题**: AcquisitionSynthesis: Targeted Data Generation using Acquisition Functions
- **作者**: Ishika Agarwal, Sofia Stoica, Emre Can Acikgoz, Pradeep Natarajan, Mahdi Namazifar, Jiaqi Ma, Dilek Hakkani-Tür (UIUC, Amazon)
- **发表日期**: 2026年5月13日
- **arXiv ID**: 2605.13149
- **arXiv 链接**: <https://arxiv.org/abs/2605.13149>
- **领域/关键词**: 主动学习, acquisition function, reward model, LLM合成数据, GRPO, 数学推理, 医学QA, 代码

## 一句话总结

> 将主动学习中的 acquisition function 作为 GRPO 强化学习的奖励信号，训练 LLM 生成器产出对学生模型最有信息量的训练数据，在数学、医学 QA 和编程任务上实现 2-7% 的分布内提升，同时比基线方法更有效地防止灾难性遗忘（OOD 性能提升 3%）。

## 论文精读

### 背景与前置知识

**主动学习 (Active Learning)**：一种机器学习策略，模型主动选择最有价值的未标注样本请求人工标注，从而在有限标注预算下最大化学习效果。关键在于"acquisition function"——衡量样本价值的函数。

**Acquisition Function (获取函数)**：主动学习中的核心概念，用于量化一个数据点对模型学习的潜在贡献。常见的策略包括不确定性采样（选择模型最不确定的样本）、多样性采样（选择与已有数据最不同的样本）等。

**拒绝采样 (Rejection Sampling)**：生成大量候选样本，然后根据某种标准筛选出高质量样本。优点是简单直接，缺点是计算效率低（生成大量样本却丢弃大部分）。

**GRPO (Group Relative Policy Optimization)**：一种强化学习算法，通过对一组生成样本进行组内比较来计算优势（advantage），不需要单独的 critic 模型。相比 PPO 更简单高效，用于训练生成策略。

**灾难性遗忘 (Catastrophic Forgetting)**：模型在学习新任务时忘记之前学过的知识。在 LLM 微调中，模型可能在某个领域性能提升，但在其他领域性能下降。

**SFT (Supervised Fine-Tuning)**：监督微调，使用标注数据对预训练模型进行有监督训练。

### 核心思想详解

想象你是一位教练，正在训练一位学生运动员。你有两种策略：

**策略 A（传统方法）**：让学生把所有时间花在健身房做固定的训练动作（固定数据集训练）。如果动作太简单，学生没有进步；太难，学生可能受伤或放弃。

**策略 B（AcquisitionSynthesis）**：教练持续观察学生的弱点和不确定之处，然后为学生设计"刚好超出当前能力"的训练任务。不太易也不太难——恰好在"最近发展区"范围内。

传统的数据生成方法要么（1）大量生成数据然后粗暴地筛选（拒绝采样），要么（2）依赖更大的模型来指出弱点（模型蒸馏）。这两种方法都缺乏一个**定量的、可解释的标准**来判断一个样本对学生的实际价值。

本论文的核心洞察：**主动学习领域的 acquisition function 正好提供了这个标准**！它衡量的是一个数据点对当前模型的"信息量"。将这些 acquisition function 作为强化学习的奖励信号，训练数据生成器产出对学生模型最有用的样本。

用图表来说明（论文 Figure 1）：未经训练的数据生成器产生的样本散落在"数据空间"各处（灰色圆圈），真正有用的很少。经过 AcquisitionSynthesis 训练后，生成器学会了集中产生位于"高 acquisition 奖励"区域的样本（绿色勾选）。

这个框架的美妙之处在于它**自洽**：同一个模型既是生成器等又是学生模型。生成器通过 GRPO 学会生成对（自己的镜像）学生模型最有用的数据——这是一种模型感知的自我改进。

### 方法逐步拆解

**步骤 1：定义 Acquisition 奖励函数**
根据主动学习文献，定义 5 种互补的奖励函数：

1. **Confidence（置信度/不确定性）**：衡量学生模型对生成样本的输出有多不确定。计算应答序列中 top-2 概率的平均差距，奖励与不确定性成反比。直觉：学生最不确定的样本能带来最大的学习信号。

2. **Proximity（邻近性）**：衡量生成样本与任务训练数据在语义空间中的距离。使用 HDBSCAN 对训练数据聚类，奖励为样本到最近聚类中心的余弦距离。直觉：太偏题的样本没有用。

3. **Gradient Magnitude（梯度幅度）**：计算生成样本通过学生模型反向传播时梯度的 Frobenius 范数。直觉：梯度越大，说明这个样本带来的"惊讶"越大，学习信号越强。

4. **Diversity（多样性）**：1 - Proximity。鼓励生成与已有训练数据不同的样本，避免模式坍缩。直觉：千篇一律的样本虽然安全但无助于泛化。

5. **Answer Variance（答案方差）**：对同一个问题用学生模型采样 k=8 次，统计不同答案的聚类数。聚类数越多，说明模型越困惑。直觉：这是语义级别的"不确定性"，而 Confidence 是 token 级别的不确定性。

此外还有一个 Format（格式）奖励：确保生成的样本符合预期的 HTML 标签格式。

**步骤 2：GRPO 训练生成器**
数据生成器（一个 LLM）使用 500 个种子样本作为上下文示例。每轮训练中：
- 生成器产生一个数据样本（问题+推理+答案）
- 将样本馈送给学生模型（生成器的相同架构但不同实例）
- 计算 5 种 acquisition 奖励
- GRPO 根据奖励信号更新生成器参数

500 个样本是关键数量——足够提供上下文示例但不会过多。

**步骤 3：生成训练数据**
训练完成后，使用生成器生成 1,000 个训练样本。

**步骤 4：训练学生模型**
在学生模型上使用 SFT 训练这些样本。

**步骤 5：评估**
在学生模型上评估 ID（分布内）和 OOD（分布外）性能。

### 关键公式/算法解读

**Confidence 计算**：
conf(P, R) = (1/|R|) Σ_t [p_θ(R_t^(1)|...) - p_θ(R_t^(2)|...)]

这是对输出序列上每个位置，计算最可能 token 概率与第二可能 token 概率之差的均值。差距越小 → 模型越不确定 → reward 越高（1/conf）。

**GRPO 训练**：
使用 GRPO（而非 PPO）的好处是：不需要单独的 critic 模型（价值函数），通过对组内生成样本的比较计算优势。数学上更简洁，训练更高效。

**数据生成的可扩展性**：
论文的关键发现之一是：**较大的生成器模型能训练出更强的生成器，而且这种能力可以迁移到不同架构的学生模型**。例如，用 Llama 8B 训练的生成器，可以为 Qwen 3B、Qwen 7B 等其他模型生成有效的训练数据。

### 实验设计分析

**为什么选这三个任务？**
- **数学（NuminaMath + AIME）**：答案可验证，有客观正确性标准
- **医学 QA（MedMCQA + PubMedQA）**：需要专业知识，考验生成相关性
- **编程（CodeForces）**：需要严谨的逻辑推理

三个任务覆盖了不同的推理类型和难度级别，提供了全面的评估。

**基线方法的选择逻辑**：
- **Original**：直接生成不调整 → 基线中的基线
- **DataEnvGym**：识别模型弱点并针对性生成 → 最直接的竞争方法
- **Prismatic Synthesis**：通过梯度聚类筛选多样数据 → 与 AcquisitionSynthesis 的 Diversity 奖励形成对比
- **Random/Filtered 数据选择**：从现有数据中选择而非生成 → 验证"生成"是否优于"选择"

**Id vs OOD 评估的双重关注**：
大多数数据生成论文只报告 ID 性能。本文同时报告 OOD 性能（注意：训练时只用了领域内数据，测试时看模型在其他任务上的表现），这是衡量灾难性遗忘的黄金标准。

**关键发现**：
1. **Confidence 是最佳通用奖励**：在所有任务和模型上表现均衡。
2. **Diversity 提供最佳成本-OOD增益平衡**：虽然总体不如 Confidence，但在 OOD 场景下性价比高。
3. **AcquisitionSynthesis 防止灾难性遗忘**：Qwen 3B 的 OOD 性能提升 3.43%，而合成基线平均下降 1.25%。
4. **跨模型迁移**：用 Llama 8B 训练的生成器可为 Qwen 3B、Qwen 7B 生成有效数据。
5. **SFT > RL > ICL**：作为训练范式比较，SFT（监督微调）在所有任务上优于 RL 和 ICL。

## 问题与动机

LLM 数据质量是模型性能的关键瓶颈。现有合成数据生成方法要么（1）靠大规模生成后拒绝采样（低效），要么（2）依赖更大的模型指导（不自主）。缺乏一个**定量的、可解释的**标准来衡量生成样本对下游学习者的价值。Acquisition function 提供了一个基于理论的数据效用度量。

## 核心方法

AcquisitionSynthesis：使用 GRPO + acquisition function 奖励训练 LLM 生成器，使其学会产出对学生模型最有信息量的训练数据。支持 5 种互补的 acquisition 奖励（置信度、邻近性、梯度幅度、多样性、答案方差），以及一个格式奖励。

## 实验结果

- **数据集**: NuminaMath（数学）、MedMCQA（医学）、CodeForces（编程）；ID 测试 + OOD 测试（AIME、PubMedQA、跨领域）
- **主要指标**: 准确率（Accuracy），ID 性能 + OOD 泛化
- **关键结果**: 
  - ID 提升 2-7%（大模型提升更大）
  - OOD 泛化：Qwen 提升 3.43%，Llama 保持稳定（+0.25%），而基线平均下降 1-13%
  - Confidence 奖励在多数任务上最佳
  - 跨模型迁移有效
- **对比方法**: Original, DataEnvGym, Prismatic Synthesis, Random, Filtered

## 局限性

1. **仅限可验证任务**：方法依赖可验证的答案（数学、医学、代码），对于开放式生成任务（如写作、对话）的适用性未验证。
2. **伪标签质量**：使用模型自生成的伪标签进行训练，标签错误可能限制性能上限（实验显示使用 GPT-4o-mini 作为标签器可提升 6%）。
3. **计算成本**：GRPO 训练 + 多 acquisition 函数计算（尤其是梯度幅度）增加了训练开销。
4. **种子数据依赖**：需要 500 个种子样本作为上下文示例，对全新任务可能需要额外收集。
5. **学生和生成器同架构**：未充分探索"弱生成器为强学生生成数据"（weak-to-strong generalization）的场景。

## 个人思考

AcquisitionSynthesis 最精彩的地方在于它**跨越了两个相对独立的领域**（主动学习和 LLM 数据生成），将主动学习中久经考验的 acquisition function 带入了一个全新的应用场景。这种"领域桥梁"式的贡献往往能产生意想不到的效果。

从哲学层面看，这解决了数据生成中的一个核心矛盾：**生成器自己不知道什么数据是有用的**（因为它不知道学生模型的状态），而 acquisition function 恰好提供了一个信号——告诉生成器学生模型在哪些地方最"饥饿"。

Confidence 作为最佳通用奖励的发现很有意思。这意味着"不确定性"可能是比"多样性"或"梯度大小"更普适的数据效用信号。但这也可能因为不确定性计算最简单、最稳定。

对于实践者来说，本文提供了一个重要的洞见：**数据生成的"目标"和"收益"可以用一个具体的信号（acquisition reward）来对齐**，而不需要依赖直觉或启发式规则。这让数据生成从"艺术"变成了"工程"。

值得思考的是：将训练和生成耦合在同一循环中是否会引入偏差？如果生成器过于擅长"针对"学生模型的弱点，会不会忽视一些虽然当前没有弱点但未来可能重要的领域？
