# MetaEarth3D: Unlocking World-scale 3D Generation with Spatially Scalable Generative Modeling

## 基本信息

- **标题**: MetaEarth3D: Unlocking World-scale 3D Generation with Spatially Scalable Generative Modeling
- **作者**: Jinqi Cao, Zhiping Yu, Baihong Lin, Chenyang Liu, Zhenwei Shi, Zhengxia Zou（北京航空航天大学）
- **发表日期**: 2026年4月19日
- **arXiv ID**: 2604.22828
- **arXiv 链接**: <https://arxiv.org/abs/2604.22828>
- **领域/关键词**: 3D generation, 行星级生成, 3D基础模型, 卫星图像, 空间可扩展

## 一句话总结
>
> MetaEarth3D 是首个能够在行星尺度进行空间一致性 3D 场景生成的生成式基础模型，通过将超广域 3D 生成重构为尺度空间和维度空间中的渐进概率分布转移，跨越地形级、城市级和街区级等多个层级，实现了从单张卫星图像或文本描述到连续无界 3D 场景的生成。

## 论文精读

### 背景与前置知识

**空间尺度（Spatial Scale）**：本文提出的核心概念——现有生成模型主要在"有界空间"内工作（如单物体、室内房间、局部街景），而 MetaEarth3D 试图将生成能力扩展到行星尺度（从 10⁻² 米到 10⁷ 米的跨度）。

**卫星图像多分辨率**：卫星图像通常以不同的空间分辨率（米/像素）提供。低分辨率图像覆盖范围广但细节少，高分辨率图像细节丰富但覆盖范围有限。MetaEarth3D 在 64m、16m、4m 和 1m/像素四种分辨率上工作。

**数字高程模型（DEM）和数字表面模型（DSM）**：DEM 表示裸土地形的高度，DSM 包含建筑物等地面物体的高度。MetaEarth3D 使用 DEM 和 DSM 作为 3D 几何的监督信号。

**马尔可夫链尺度转移**：假设高分辨率图像仅依赖于同一区域对应的低分辨率图像，将多尺度生成分解为条件概率的链式乘积。这使得单个网络可以通过参数复用来处理所有尺度的生成任务。

**无界生成（Unbounded Generation）**：由于 GPU 内存限制，超广域图像不能一次性生成。MetaEarth3D 采用滑动窗口分块生成，并通过确定性 ODE 求解器（DDIM）保证相邻窗口重叠区域的像素一致。

### 核心思想详解

MetaEarth3D 的核心洞察可以用一个思想实验来理解：

**假设你想生成整个地球的 3D 模型。** 传统的方法要么只生成一个小房间（室内场景生成），要么生成一个特定视角的城市景观（如 CityDreamer）。但如果你想生成从太空俯瞰的整个山脉、放大到城市、再放大到街区，现有方法就无能为力了。

MetaEarth3D 的方法是：**不要试图一次性生成所有内容，而是将生成过程分解为两个维度的渐进过渡——"尺度空间"和"维度空间"。**

**尺度空间**处理的是"从粗到细"的问题：先在一个 64 米/像素的极低分辨率下生成一个大地形的基本布局（山在哪、城市在哪），然后逐步提高分辨率到 16m、4m、1m/像素，每一步都在前一步的基础上添加细节。这就像一个画家先画轮廓，再画大色块，最后才画细节纹理。

**维度空间**处理的是"从 2D 到 3D"的问题：先生成 2D 卫星图像（从俯视图看的地面纹理），然后基于 2D 图像预测高程图（地形和建筑物的高度），最后生成侧面的纹理（如建筑物立面）。这样就将一个极其复杂的 3D 生成任务分解为三个较简单的 2D 条件生成子任务。

**另一个关键思想是"无界生成"**：由于无法将整个上海一次性塞进 GPU，MetaEarth3D 用滑动窗口分块生成，然后通过一个巧妙的技巧保证块之间无缝拼接——在相邻窗口的重叠区域使用完全相同的初始噪声，并采用确定性 ODE 求解器（而不是随机 SDE），保证相同输入产生相同输出，从而实现像素级连续。

### 方法逐步拆解

#### 第一部分：渐进概率框架

**步骤 1：概率分解**。将超广域 3D 场景 X 的概率分布分解为条件链：
- p(X) = p(x_o) · p(x_h | x_o) · p(x_l | x_o, x_h)
- x_o = 正射影像（卫星俯视图）
- x_h = 高程图（地形和建筑物高度）
- x_l = 侧面外观（建筑物立面纹理）

**步骤 2：尺度空间中的递进式生成**。将多尺度卫星图像生成建模为马尔可夫链：
- p(x_o^(K), ..., x_o^(k)) = p(x_o^(k)) · Π p(x_o^(i+1) | x_o^(i), s^(i+1))
- 从低分辨率（64m/像素）逐步向高分辨率（1m/像素）过渡
- 使用统一网络参数，通过目标分辨率嵌入 s^(i+1) 控制生成行为
- 这利用了地理空间特征的自相似性——相邻分辨率场景高度相关

#### 第二部分：递归多尺度卫星图像生成模块

**递归扩散网络**：
- 采用 DDPM 骨干网络（约 1B 参数）
- 低分辨率图像 x_o^(i) 通过图像编码器提取特征，与目标分辨率嵌入一起作为条件
- 目标分辨率通过正弦位置编码嵌入，与时间步嵌入一同通过自适应归一化与网络特征交互
- 训练标准噪声预测损失

**无界生成算法**：
- 采用 DDIM 确定性采样（概率流 ODE）而非随机 SDE
- 滑动窗口 50% 重叠遍历场景
- 重叠区域使用完全相同的初始噪声 x_T
- 由于 ODE 求解器的确定性，相同初始噪声 + 相同条件 = 相同输出
- 理论上保证像素级无缝拼接

#### 第三部分：几何-纹理解耦的维度提升模块

**3.1 生成式高度推断（2D → 2.5D）**：
- 将高度估计重构为文本提示条件图像到图像翻译任务
- 提示为"预测显著特征的高度"
- 输入 x_o 通过 VAE 编码，与带噪声的高程隐变量拼接
- 扩散网络学习从视觉特征映射到几何结构
- 无界生成扩展到隐空间：滑动窗口 + 一致初始化噪声

**3.2 多视角侧面外观修补（2.5D → 3D）**：
- 将预测的高度图与正射影像通过几何投影合成粗 3D 网格
- 垂直表面（建筑物立面）缺乏纹理，需要生成
- 设计多视角纹理联合生成扩散模块：
  - 从 8 个等距视角渲染粗网格，识别缺失纹理区域（通过表面法线分析）
  - 对每个视角，输入为：带噪声的隐变量 + VAE 编码的粗渲染 + 缺失掩码
  - 显式相机姿态注入：视角索引映射为可学习嵌入，加到输入卷积特征上，打破序列置换不变性
  - **跨视角局部注意力**：每个视角的查询只关注自身和相邻视角（i-1, i, i+1，环状索引）的键和值。相邻视角有显著视觉重叠，局部注意力比全局注意力更高效且减少噪声

**3.3 纹理反投影**：
- 通过逆投影和可见性感知的纹理烘焙将生成的 2D 多视角图像映射回 3D 网格
- 使用 Z-buffer 做可见性检测，选择视角方向与表面法线最对齐的视图采样纹理
- 水平表面保留原始正射纹理，垂直表面使用生成纹理

#### 数据集构建

MetaEarth3D 构建了约 1000 万张图像的大规模数据集：
- **多尺度卫星图像**（~500 万张）：从公开遥感数据集收集，4 种分辨率（64m、16m、4m、1m/像素），采用 GPT-4 自动标注+人工抽检，覆盖城市、森林、沙漠、冰川等
- **高程图**（~120 万张）：与卫星图像地理配准，包含 16m/像素 DEM（全球）和 1m/像素 DSM（79 个主要城市，~8000 km²）
- **多视角城市图像**（~100 万张）：在 Google Earth Studio 中为 116 个全球城市设置圆形轨迹，每个轨迹 40 个视角，捕获建筑物立面纹理

### 关键公式/算法解读

**整体概率分解**：
```
p(x_o, x_h, x_l) = p(x_o) · p(x_h|x_o) · p(x_l|x_o, x_h)
```
将 3D 场景生成解耦为三个条件生成步骤：正射影像 → 高程图 → 侧面纹理。

**马尔可夫链尺度转移**：
```
p(x_o^(K), ..., x_o^(k)) = p(x_o^(k)) · Π p(x_o^(i+1) | x_o^(i), s^(i+1))
```
多尺度生成分解为从低分辨率到高分辨率的条件递进，每步只依赖前一步的输出和目标分辨率 s。

**DDIM 确定性采样**：
```
p_θ1(x_(t-1)^(i+1) | x_t^(i+1), c^(i+1)) = N(x_(t-1)^(i+1); μ_θ, σ_t²I)
```
使用确定性 ODE 求解器而非随机 SDE，保证相同初始噪声产生相同输出，实现无界生成的无缝拼接。

**跨视角局部注意力**：
```
Attention(q_i, K, V) = Softmax(q_i · K_N(i)ᵀ / √d) · V_N(i)
```
每个视角的查询只关注自身和相邻两个视角的键和值，建模共享纹理模式。

### 实验设计分析

**评估指标**：
- **FID**：衡量生成图像质量（越低越好），在 Manhattan/Brooklyn 区域与 CityDreamer、GaussianCity 对比
- **MSG（平均接缝梯度）**：衡量无界生成的连续性（越低越好）
- **PSNR/SSIM**：衡量多视角纹理一致性
- **高度分布统计**：对比生成与真实高程的均值和标准差
- **下游任务性能**：在 5 种地理空间理解任务上评估作为数据引擎的效果

**关键结果**：

1. **生成质量**：FID 76.04（CityDreamer 86.94，-12.5%），超越现有 SOTA。50 名领域专家 5 维度评估（感知质量、3D 真实感、多样性、视角一致性、空间尺度）均获得 4.1-4.9 高分。

2. **无界连续性**：完整方法（滑动窗口+重叠+一致噪声）的 MSG 显著低于消融基线（无重叠、无噪声约束），说明无界生成算法的有效性。

3. **多视角一致性**：PSNR 28.64 dB（基线 27.11），SSIM 0.9076（基线 0.8909），跨视角局部注意力和姿态注入有效改善了多视角纹理一致性。

4. **统计真实性**：
   - 生成卫星图像与真实图像的语义特征分布 (t-SNE) 紧密重叠，FID 24.51（DiffusionSat 27.87）
   - 生成高程图在全球 6 大洲的分布直方图与真实数据高度一致（均值、标准差、分布形态匹配），包括复杂非高斯和多模态分布

5. **作为数据引擎**：利用 MetaEarth3D 生成 60 场景/7690 样本的合成训练数据，微调 2D VLM 在 5 种地理空间理解任务（空间、形态、计数、几何、描述）上平均提升 +22.85%，验证了生成数据的实际应用价值。

6. **效率**：单张 RTX 4090（24GB）上生成 17 km² 场景仅需 2 小时。

## 问题与动机

现有生成模型在空间尺度上受到根本性限制——只能生成有界环境内的内容（物体级、室内级、局部街景级）。这一限制对超广域空间智能（远程监控、航空自主导航、灾害管理）构成了关键瓶颈。将视觉生成扩展到行星尺度面临三大挑战：（1）超广域环境的统一表示；（2）高效且空间一致的场景表示；（3）极端训练复杂度。

## 核心方法

MetaEarth3D 将超广域 3D 生成重构为渐进概率转移框架：**尺度空间**中通过自级联递归扩散网络实现从低分辨率到高分辨率的逐级细化（统一参数、空间分辨率嵌入控制）；**维度空间**中通过几何-纹理解耦实现 2D→2.5D→3D 的递进提升——先预测正射影像，再推断高程图，最后生成多视角侧面纹理。采用确定性 DDIM 采样和一致初始噪声实现无界拼接。构建约 1000 万张图像的全球多尺度数据集。

## 实验结果

- **数据集**: 约 1000 万张图像（多尺度卫星图像 ~500 万 + 高程图 ~120 万 + 多视角城市图像 ~100 万），全球分布
- **主要指标**: FID、MSG（平均接缝梯度）、PSNR/SSIM（多视角一致性）、高程分布统计、下游任务准确率
- **关键结果**: FID 76.04（SOTA，-12.5%）；专家评估 4.1-4.9/5；高程分布与真实数据高度匹配；作为数据引擎提升下游任务 +22.85%；单卡 RTX 4090 生成 17 km² 场景仅 2 小时
- **对比方法**: CityDreamer、GaussianCity、InfiniCity（3D 城市生成）；DiffusionSat（卫星图像生成）

## 局限性

1. **高度估计的歧义性**：从单视图卫星图像推断高度本质上是病态问题，在视觉线索不足时可能产生不合理的高程。
2. **侧面纹理合成的简化假设**：多视角纹理生成基于固定轨迹，对于复杂几何结构（如悬挑、拱形）可能无法完全正确地生成所有侧面。
3. **受限于卫星视角**：主要面向俯视和斜视场景，不适用于地面第一人称视角的 3D 场景生成。
4. **模型规模大**：46 亿参数，虽然推理可以在单卡上进行，但训练需要大规模计算资源。
5. **时序动态缺失**：当前生成的是静态场景，不支持时间变化（如光照变化、植被季节变化、建筑动态）。
6. **图像质量多样性差距**：在城市细节区域，生成图像在某些较难案例上仍与真实数据存在可辨别的质量差距。

## 个人思考

MetaEarth3D 是"空间尺度作为智能核心维度"这一论点的有力论证。它挑战了生成模型研究的一个基本假设——更大的模型和更多的数据就是一切。作者指出，除了参数和数据规模外，**空间可扩展性**是基础模型应该考虑的另一个轴。这对遥感、地理信息系统、智慧城市等领域具有深远意义。

技术上最打动我的是"无界生成"的巧妙设计：用 DDIM 的确定性采样特性 + 一致的初始噪声，数学上保证滑动窗口的无缝拼接。这个"trick"简单、优雅、有效，比复杂的后处理拼接方法优雅得多。

作为"数据引擎"的潜力令人兴奋。MetaEarth3D 生成的场景自带精确的 3D 标注（高度、空间关系等），这对于训练地理空间 AI 模型来说极其珍贵——真实世界的 3D 标注数据获取成本极高。文中用合成的 7690 个样本就提升了 VLM 在 5 种任务上 22.85% 的性能，证明了合成数据的巨大价值。

一个可能的有趣方向是将 MetaEarth3D 与室内场景生成方法（如 HetScene、CasLayout）结合——从行星尺度到城市尺度再到室内尺度，实现真正的多级可缩放场景生成，覆盖从太空到桌面的完整尺度谱系。
