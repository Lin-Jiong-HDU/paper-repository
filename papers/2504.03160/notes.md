## 一句话总结
>
> DeepResearcher 是首个在真实网络搜索环境中通过端到端强化学习训练 LLM 深度研究智能体的框架，相比 prompt 工程基线提升高达 28.9 分，并涌现出规划、交叉验证、反思和诚实等类人认知行为。

## 论文精读

### 背景与前置知识

理解这篇论文需要以下基础概念：

- **大型语言模型（LLM）**：如 GPT、Qwen 等大规模预训练模型，能够理解和生成自然语言。
- **RAG（检索增强生成）**：一种让 LLM 在回答问题时先检索外部文档、再基于检索结果生成答案的方法。传统 RAG 使用固定的本地知识库（如 Wikipedia 文本），假设所有需要的信息都已在库中。
- **Prompt Engineering**：通过精心设计提示词来引导 LLM 的行为。这种方法灵活但脆弱，难以泛化到复杂任务。
- **强化学习（RL）**：一种机器学习范式，智能体通过与环境的交互获得奖励信号，从而学习最优策略。DeepSeek-R1 和 Kimi K1.5 等工作已证明 RL 能显著提升 LLM 的推理能力。
- **GRPO（Group Relative Policy Optimization）**：一种 RL 算法，通过同一问题生成多条轨迹（rollout），用组内相对表现来估计优势函数，无需训练额外的 critic 网络。
- **深度研究（Deep Research）**：指需要多步搜索、信息综合和推理的复杂研究任务，如 OpenAI Deep Research 等产品所展示的能力。

### 核心思想详解

现有的深度研究智能体存在两个关键问题：**基于 prompt 工程的方法**行为僵化、泛化能力差；**基于 RAG 的 RL 训练方法**（如 Search-R1、R1-Searcher）虽然在静态知识库中训练，但无法应对真实网络的噪声、多样性和动态性。

DeepResearcher 的核心洞察是：**在真实网络环境中进行端到端 RL 训练不是锦上添花，而是开发鲁棒研究能力的根本要求。** 就像一个学生只在模拟题中练习（RAG 环境）永远无法完全应对真实考试（开放网络）一样——真实环境中的信息可能是过时的、矛盾的、或根本不存在的。

类比来说，如果 RAG 训练是在"温室"中培育植物，那么 DeepResearcher 就是在"野外"环境中培育——虽然更艰难，但长出来的能力更强壮、更适应真实世界。

### 方法逐步拆解

1. **智能体轨迹设计**：DeepResearcher 采用迭代式的"推理-行动"循环。每一步先在 `<think />` 标签中进行推理，然后选择工具调用（网络搜索或网页浏览），获取观察结果后再进入下一轮推理，直到有足够信息生成最终答案。

2. **多智能体架构**：系统包含两类智能体——主研究智能体负责推理和决策，浏览智能体专门负责从网页中提取相关信息。浏览智能体会将长网页分段处理，维护短期记忆，按顺序阅读并在信息不足时决定继续阅读还是停止。这种设计模拟了人类研究员的行为：先快速扫视，再深入阅读。

3. **真实环境中的工程挑战处理**：
   - **高并发请求**：GRPO 每步需要大量采样（如 4096 个并发搜索请求），为此搭建了 50 节点的分布式 CPU 服务器集群来处理。
   - **反爬虫与 API 限制**：实现重试机制和搜索结果缓存（7 天内相同查询复用结果），既应对限制又降低成本。
   - **信息提取优化**：浏览智能体按段落顺序阅读，如果前几段就明显不相关则跳过该网页，提高效率。

4. **RL 训练框架**：采用 GRPO 算法，每步采样 256 个 prompt，每个 prompt 生成 16 条轨迹（rollout），每条轨迹最多 10 次工具调用。关键地，工具返回的观察结果被 mask 掉，不参与梯度计算——只有模型自身生成的 token 参与。

5. **奖励设计**：使用 F1 分数作为主要奖励指标。如果输出格式错误（如缺少标签），给予 -1 的惩罚；格式正确则奖励为预测答案与标准答案的 F1 分数。

6. **数据去污染**：为防止模型依赖记忆而非搜索来回答问题，对每个训练问题从基础模型采样 10 次回答，如果任一次包含正确答案则排除该问题。最终构建了 80,000 个训练样本，其中多跳问题占 75%（NQ:TQ:HotpotQA:2Wiki = 1:1:3:3）。

### 关键公式/算法解读

**GRPO 目标函数**：给定输入 x 和一组轨迹 τ = {y_i}，GRPO 的优化目标为：

- 对每条轨迹，计算当前策略与旧策略的概率比 r_i = π_θ(y_i|x) / π_θ_old(y_i|x)
- 使用 clip 操作将比率限制在 [1-ε, 1+ε] 范围内，防止策略更新过大
- 优势函数 A_i 基于组内轨迹的相对表现估计（无需 critic 网络）
- 加入 KL 散度惩罚项 β·D_KL(π_θ || π_θ_ref)，防止偏离参考策略太远
- 最终目标是在所有轨迹上取平均

**奖励函数**：

- 格式错误 → reward = -1
- 格式正确 → reward = F1 score（预测答案与标准答案在词级别的 F1 分数）

**观察遮蔽（Observation Masking）**：工具返回的观察内容不参与损失计算，只有模型自身生成的 token 计入梯度。这确保模型学习的是如何推理和决策，而非记忆搜索结果。

### 实验设计分析

**为什么选这些数据集**：实验使用了 7 个开放域 QA 数据集，分为域内（NQ、TQ、HotpotQA、2Wiki）和域外（MuSiQue、Bamboogle、PopQA）。域内数据集是训练数据的来源，测试模型是否学好了；域外数据集考察泛化能力。特别地，Bamboogle 的语料不限于 Wikipedia，能测试模型在非 Wikipedia 知识上的表现。

**为什么用 F1 和 MBE 双指标**：F1 是基于词匹配的规则指标，MBE（Model-Based Evaluation）使用 GPT-4o-mini 作为评判者，能处理语义等价但措辞不同的情况。两者结合更全面地评估答案质量。

**结果说明了什么**：
- DeepResearcher 在所有 7 个数据集的 MBE 指标上都是最优的
- 相比 prompt 工程基线（Search-o1 + Web Search）提升高达 28.9 分（Bamboogle）
- 相比 RAG-based RL（R1-Searcher）提升高达 7.2 分（Bamboogle）
- 即使 R1-Searcher 也在网络环境中推理，DeepResearcher 仍显著优于它，证明**训练**环境的真实性才是关键
- 在 Bamboogle 上优势最大，因为该数据集需要的知识超出 Wikipedia 范围

## 问题与动机

现有的深度研究智能体要么依赖人工设计的 prompt（脆弱、难泛化），要么在静态 RAG 环境中训练（无法应对真实网络的复杂性）。OpenAI 的 Deep Research 使用了 RL 技术，但方法细节完全保密。DeepResearcher 旨在填补这一空白，在真实的开放网络环境中通过 RL 端到端训练研究智能体。

## 核心方法

DeepResearcher 采用多智能体架构：主智能体进行推理和工具选择，浏览智能体负责从网页中增量式提取信息。训练使用 GRPO 算法，在 50 节点分布式集群上运行，通过搜索缓存和重试机制应对真实网络的工程挑战。数据方面，通过去污染筛选构建了 80,000 个训练样本，强调多跳推理场景。

## 实验结果

- **数据集**: NQ, TriviaQA, HotpotQA, 2WikiMultiHopQA（域内）；MuSiQue, Bamboogle, PopQA（域外）
- **主要指标**: F1 Score, Model-Based Evaluation (MBE)
- **关键结果**: 在所有 7 个数据集上 MBE 最优；相比 prompt 基线最高提升 28.9 分；相比 RAG-based RL 最高提升 7.2 分
- **对比方法**: CoT Only, CoT + RAG, Search-o1, Search-r1 (base/instruct), R1-Searcher

## 局限性

1. **短答案评估的局限**：当前使用 F1 分数作为奖励，只适用于短答案的 QA 任务。对于需要长文本回答的深度研究报告，需要更复杂的奖励机制。
2. **计算成本高昂**：在真实网络环境中训练需要大量搜索 API 调用和网页抓取，搭建了 50 节点集群，成本和工程复杂度都很高。
3. **基础模型的依赖**：使用 Qwen2.5-7B-Instruct 作为基础模型，未探索更大模型的效果。
4. **工具调用的限制**：当前固定使用 top-k=10 的搜索结果，未探索动态参数优化。
5. **诚实的评估问题**：模型学会了在找不到答案时坦承不知，但现有的评估指标并未奖励这种行为。

## 个人思考

DeepResearcher 的核心贡献在于证明了"真实环境训练"的重要性——这不仅仅是一个工程细节，而是影响模型能力本质的关键因素。这类似于机器人学习中 sim-to-real gap 的问题：在模拟器中学到的策略往往在真实世界中失效。

一个值得关注的现象是**涌现的认知行为**：规划、交叉验证、反思和诚实这些行为都不是通过 SFT 显式教导的，而是通过端到端 RL 自然涌现的。这暗示 RL 训练可能是一种更高效地培养复杂认知行为的方式。

不过，当前方法仍局限于短答案 QA 场景。真正的深度研究需要处理长文本报告、多文档综合分析、矛盾信息处理等更复杂的能力。如何将这一框架扩展到这些场景，是一个值得探索的方向。
