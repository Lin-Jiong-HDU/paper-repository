## 基本信息

- **标题**: Direct Preference Optimization: Your Language Model is Secretly a Reward Model
- **作者**: Rafael Rafailov, Archit Sharma, Eric Mitchell, Stefano Ermon, Christopher D. Manning, Chelsea Finn
- **发表日期**: 2023年5月
- **arXiv ID**: 2305.18290

- **领域/关键词**: RLHF, Preference Optimization, LLM Alignment, DPO, Bradley-Terry Model, Reward Model

## 一句话总结
>
> DPO 通过将 RLHF 中的奖励模型隐式地表达为语言策略本身，把复杂的"训练奖励模型 + 强化学习优化"两阶段流程简化为一个简单的二元分类损失，无需强化学习即可直接从人类偏好数据中优化语言模型。

## 论文精读

### 背景与前置知识

**大语言模型的对齐问题。** 大规模无监督语言模型（LM）在大量文本上训练后获得了广泛的世界知识和一定的推理能力，但要精确控制其行为却非常困难。训练数据来自各种目标、优先级和技能水平的用户，模型可能模仿不良行为。因此，如何从模型的广泛知识和能力中"选择"出安全、高效、可控的行为模式，是构建 AI 系统的关键问题。

**强化学习从人类反馈（RLHF）。** 当前最成功的对齐方法族是 RLHF（Reinforcement Learning from Human Feedback）。标准 RLHF 流程分为三个阶段：

1. **监督微调（SFT）**：用高质量数据对预训练语言模型进行监督微调，得到基础模型 pi_SFT。
2. **偏好采样与奖励学习**：用 SFT 模型对提示生成成对回答，由人类标注偏好（yw > yl），然后训练一个奖励模型 r_phi 来拟合这些偏好数据。
3. **强化学习优化**：使用 RL（通常为 PPO）来最大化奖励模型的输出，同时用 KL 散度约束防止策略偏离参考模型太远。

**Bradley-Terry（BT）偏好模型。** RLHF 中常用的偏好建模框架。给定两个回答 y1, y2 和提示 x，BT 模型假设人类偏好分布可以写成：

- p*(y1 > y2 | x) = exp(r*(x, y1)) / (exp(r*(x, y1)) + exp(r*(x, y2)))

即偏好概率与奖励值的指数成正比。奖励模型通过最大似然估计来训练，损失函数是负对数似然。

**KL 约束的奖励最大化。** RLHF 的 RL 阶段目标为：

- max E[r_phi(x, y)] - beta * D_KL(pi_theta || pi_ref)

其中 beta 控制策略偏离参考模型的程度。这个约束很重要：它防止模型偏离奖励模型准确的有效分布区域，同时保持生成多样性，避免模式坍缩到单一高奖励回答。

**RLHF 的痛点。** 虽然 RLHF 效果显著，但流程极其复杂：需要训练多个模型（SFT 模型、奖励模型、策略模型、价值网络），需要在训练循环中从 LM 策略采样，计算成本高昂，且 RL 训练本身不稳定。

### 核心思想详解

DPO 的核心洞察是：**语言模型本身就隐式地包含了一个奖励函数。** 具体而言，如果我们知道 KL 约束奖励最大化问题的最优策略的解析形式，就可以将奖励函数用策略和参考策略来表达，从而把"拟合奖励模型"的问题转化为"直接优化策略"的问题。

**关键步骤的直觉理解：**

1. **从奖励到策略的映射。** KL 约束奖励最大化问题的最优策略有解析解：最优策略 pi_r 正比于参考策略 pi_ref 乘以奖励的指数。这意味着给定任何奖励函数，我们都可以直接写出对应的最优策略。

2. **从策略到奖励的逆映射。** 更关键的是，这个关系可以反转——给定最优策略 pi_r 和参考策略 pi_ref，我们可以用对数概率比来恢复奖励函数（加上一个仅与提示 x 有关的配分函数项）。

3. **配分函数的消除。** BT 偏好模型只依赖于两个回答的奖励差，因此配分函数 Z(x) 在计算偏好概率时会相互抵消。这意味着我们可以完全用策略的概率比来表达人类偏好概率，不需要知道 Z(x)。

4. **直接优化策略。** 既然偏好概率可以直接用策略表达，那么我们就不需要先训练奖励模型再优化策略。取而代之，我们可以直接用最大似然来训练策略，使其产生的偏好概率与人类标注一致。这就是 DPO 的损失函数——一个简单的二元交叉熵损失。

**与朴素方法的区别。** 朴素地增加偏好回答的概率、降低非偏好回答的概率（即 Unlikelihood 方法）会导致模型退化。DPO 的关键改进在于引入了一个动态的、按样本调整的重要性权重：当隐式奖励模型错误地将非偏好回答排在偏好回答之上时，该样本会获得更高的权重。这个权重自动调节了优化的力度。

### 方法逐步拆解

**第一步：建立 KL 约束优化问题的解析解。**

从标准 RLHF 目标出发：max E[r(x, y)] - beta * KL(pi || pi_ref)。通过引入配分函数 Z(x) = sum_y pi_ref(y|x) exp(r(x,y)/beta)，可以证明最优策略为：

- pi*(y|x) = (1/Z(x)) * pi_ref(y|x) * exp(r(x,y)/beta)

**第二步：反转奖励与策略的关系。**

对上述解析解取对数并重新整理，得到：

- r(x, y) = beta * log(pi*(y|x) / pi_ref(y|x)) + beta * log Z(x)

即奖励可以用最优策略和参考策略的对数比来表达。

**第三步：代入 BT 偏好模型并消去配分函数。**

将上面的奖励表达式代入 BT 模型 p*(y1 > y2 | x) = sigma(r*(x,y1) - r*(x,y2))，配分函数 beta * log Z(x) 在做差时抵消，得到：

- p*(y1 > y2 | x) = sigma(beta * log(pi*(y1|x)/pi_ref(y1|x)) - beta * log(pi*(y2|x)/pi_ref(y2|x)))

**第四步：构造最大似然目标。**

现在偏好概率完全用策略表达，我们可以对参数化策略 pi_theta 构造最大似然目标（即负对数似然损失）：

- L_DPO(pi_theta; pi_ref) = -E_{(x,yw,yl)~D} [log sigma(beta * log(pi_theta(yw|x)/pi_ref(yw|x)) - beta * log(pi_theta(yl|x)/pi_ref(yl|x)))]

这就是 DPO 的最终损失函数——一个标准的二元交叉熵损失。

**第五步：梯度分析理解优化行为。**

DPO 损失的梯度为：

- nabla_theta L_DPO = -beta * E[sigma(r_hat(x, yl) - r_hat(x, yw))] * [nabla_theta log pi(yw|x) - nabla_theta log pi(yl|x)]

其中 r_hat(x, y) = beta * log(pi_theta(y|x) / pi_ref(y|x)) 是隐式奖励。梯度同时增加偏好回答的概率、降低非偏好回答的概率，且按隐式奖励模型"犯错的严重程度"加权。

### 关键公式/算法解读

**公式 (4)：最优策略的解析解。** pi*(y|x) = (1/Z(x)) * pi_ref(y|x) * exp(r(x,y)/beta)。这是 KL 约束奖励最大化问题的闭式解，表明最优策略是参考策略按奖励重新加权后的归一化分布。Z(x) 是配分函数，保证这是一个有效的概率分布。

**公式 (5)：奖励的重参数化。** r(x, y) = beta * log(pi*(y|x) / pi_ref(y|x)) + beta * log Z(x)。这个关键等式建立了从策略到奖励的逆向映射，是 DPO 方法的理论基础。

**公式 (6)：用策略表达的偏好概率。** p*(y1 > y2 | x) = sigma(beta * log(pi*(y1|x)/pi_ref(y1|x)) - beta * log(pi*(y2|x)/pi_ref(y2|x)))。配分函数项在做差时被消去，使得偏好概率仅依赖策略。

**公式 (7)：DPO 损失函数（核心公式）。** L_DPO = -E[log sigma(beta * log(pi_theta(yw|x)/pi_ref(yw|x)) - beta * log(pi_theta(yl|x)/pi_ref(yl|x)))]。这是 DPO 的完整目标函数，输入是偏好数据对 (x, yw, yl)，输出是一个标量损失。

**DPO 算法流程：**
1. 对每个提示 x，从参考策略采样两个回答 y1, y2
2. 由人类标注偏好，构建离线偏好数据集 D
3. 初始化 pi_ref = pi_SFT（或用偏好回答微调得到）
4. 优化 pi_theta 以最小化 L_DPO

### 实验设计分析

**为什么选择这三个任务？**

- **受控情感生成（IMDb）**：使用预训练情感分类器作为真实奖励函数，可以精确计算 reward-KL 前沿，从而在受控环境下公平比较不同算法的优化效率。选择这个任务是因为它允许我们回答核心问题：DPO 在优化 RLHF 目标方面有多高效？
- **文本摘要（Reddit TL;DR）**：这是 RLHF 领域的经典基准任务，已有标准的人类偏好数据集（Stiennon et al.），且先前工作已证明 PPO 在此任务上优于 SFT。选择它验证 DPO 在真实偏好数据上的可扩展性。
- **单轮对话（Anthropic HH）**：对话是 LLM 对齐的核心应用场景。这个任务使用 Anthropic 的 Helpful and Harmless 数据集（170k 对话），测试 DPO 在更大规模、更开放任务上的表现。

**结果说明了什么？**

- **Reward-KL 前沿（情感任务）**：DPO 在所有 KL 值下都取得了最高的期望奖励，其前沿严格优于 PPO，甚至优于使用真实奖励的 PPO-GT（oracle）。这说明 DPO 不仅简化了流程，优化效率也更高。
- **摘要任务**：DPO 在温度 0.0 时的胜率为 61%，超过了 PPO 的 57%。DPO 对采样温度更鲁棒——PPO 在高温时性能退化到基线水平，而 DPO 保持稳定。
- **对话任务**：DPO 是唯一一个能稳定超越数据集中偏好回答的计算高效方法，且与计算代价高昂的 Best-of-128 基线性能相当。
- **分布外泛化**：将摘要模型应用于 CNN/DailyMail 新闻文章，DPO 仍然以显著优势超越 PPO（36% vs 26%），表明 DPO 策略具有较好的泛化能力。
- **人类评估验证**：人类评估与 GPT-4 评估一致性较高（70%-86%），DPO 样本在人类评估中以 58% 的胜率超越 PPO 样本。

## 问题与动机

RLHF 虽然有效但流程极其复杂：需要训练多个模型（SFT、奖励模型、策略模型、价值网络），需要在训练循环中从语言模型策略采样，涉及大量的超参数调优，且 RL 训练本身不稳定（如 PPO 中奖励归一化、价值函数估计等问题）。作者的核心动机是：能否绕过显式的奖励模型训练和 RL 优化，直接从偏好数据中学习最优策略？

## 核心方法

DPO 的核心是一种变量替换策略（change of variables）：

1. 利用 KL 约束奖励最大化问题的解析解，建立奖励函数与最优策略之间的双向映射
2. 将奖励函数用策略的对数概率比重参数化
3. 在 Bradley-Terry 偏好模型中，配分函数在做差时自然消去
4. 因此偏好概率可以完全用策略表达，无需显式训练奖励模型
5. 直接对策略施加最大似然目标，即一个简单的二元交叉熵损失

理论分析（Theorem 1）进一步证明：这种重参数化并不限制可表示的奖励模型类别——所有与 Plackett-Luce/BT 模型一致的奖励等价类都可以用这种方式表示。

## 实验结果

- **数据集**: IMDb (情感生成), Reddit TL;DR (摘要), Anthropic HH (对话), CNN/DailyMail (泛化测试)
- **主要指标**: Reward-KL 前沿, GPT-4 胜率, 人类评估胜率
- **关键结果**: DPO 在所有任务上匹配或超越 PPO；Reward-KL 前沿严格优于 PPO（含 oracle）；摘要胜率 61% vs PPO 57%；人类评估 DPO 以 58% 胜 PPO；分布外泛化也优于 PPO
- **对比方法**: PPO, PPO-GT (oracle), Preferred-FT, Unlikelihood, Best-of-N, SFT, GPT-J zero-shot, Pythia-2.8B 2-shot

## 局限性

- 论文仅评估了最大 6B 参数的模型，DPO 是否能扩展到更大规模的模型（如 70B+）尚未验证
- DPO 策略的分布外泛化能力需要更系统的研究，初步结果虽积极但范围有限
- 奖励过度优化（reward over-optimization）在 DPO 中如何体现尚不清楚，Figure 3 右图中出现的轻微性能下降可能是其征兆
- GPT-4 评估的胜率受提示词影响（论文发现简洁性提示比简单提示更接近人类判断）
- 理论上 DPO 重参数化选择了等价类中满足配分函数归一化的特定奖励函数，但这种选择是否有实际影响未深入探讨
- DPO 依赖离线偏好数据，无法像在线 RLHF 那样在训练过程中利用未标注的提示进行自我标注

## 个人思考

DPO 是 LLM 对齐领域的一个里程碑式工作，其优雅之处在于用一个简单的数学技巧（变量替换）将复杂的 RL 问题转化为简单的分类问题。这种"化繁为简"的思路值得学习。从实践角度看，DPO 大幅降低了 RLHF 的门槛——不需要 RL 工程经验、不需要复杂的采样和价值网络，一个简单的分类损失就能实现同等甚至更好的效果。

DPO 的成功也揭示了一个有趣的事实：语言模型在训练过程中本身就编码了某种"奖励信号"，只是我们之前没有找到正确的方式来提取和利用它。这个洞察可能对更广泛的 AI 对齐研究有启发意义。

后续工作（如 IPO、KTO、ORPO 等）在 DPO 基础上进一步探索了不同的偏好建模方式和训练策略，表明 DPO 开辟的研究方向具有持续的影响力。
