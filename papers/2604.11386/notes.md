# ComSim: Building Scalable Real-World Robot Data Generation via Compositional Simulation

## 基本信息

- **标题**: ComSim: Building Scalable Real-World Robot Data Generation via Compositional Simulation
- **作者**: Yiran Qin, Jiahua Ma, Li Kang, Wenzhan Li, Yihang Jiao, Xin Wen, Xiufeng Song, Heng Zhou, Jiwen Yu, Zhenfei Yin, Xihui Liu, Philip Torr, Yilun Du, Ruimao Zhang

## 一句话总结

> ComSim 提出了一种"组合仿真"新范式，通过融合经典仿真（精确的动作-视频配对）和神经仿真（真实感渲染），构建了一个闭环的 "真实→仿真→真实"数据增强流水线，仅需少量真实数据就能生成大规模、多样化且动作一致的训练数据。

## 论文精读

### 背景与前置知识

**经典仿真（Classical Simulation）** 使用物理引擎（如 MuJoCo、Isaac Lab、ManiSkill）来模拟机器人与环境的交互。其优势在于可以精确记录每个时刻的机器人动作和对应的视觉观测，生成完美的"动作-视频"配对数据。缺点是渲染画面与真实世界存在视觉差距，且物理模拟不够真实。

**神经仿真（Neural Simulation）** 使用视频生成模型（如扩散模型）来根据动作序列生成对应的视频。其优势是可以生成与真实世界视觉一致的视频，但存在"幻觉"问题——生成的视频可能在物理上不一致（例如物体穿过桌子、机器人手指穿过物体），导致动作与视频之间的对应关系被破坏。

**领域鸿沟（Domain Gap）** 是 sim-to-real 的核心问题。经典仿真和神经仿真各自擅长解决鸿沟的不同方面：经典仿真保证了动作的精确性但视觉不真实；神经仿真保证了视觉的真实性但动作控制不精确。

**扩散模型（Diffusion Model）** 是一类生成模型，通过逐步向数据添加噪声（前向过程）然后学习反向去噪来生成新样本。DiT（Diffusion Transformer）是将 Transformer 架构应用于扩散模型的变体。

**控制动态和视觉动态**：控制动态指机器人的动作序列（如关节角度、末端执行器位姿）；视觉动态指摄像头观测到的图像序列。

### 核心思想详解

ComSim 的核心洞察非常直观：为什么要在经典仿真和神经仿真之间二选一？为什么不把两者的优势组合起来？

**类比**：想象你要为一个电影场景制作特效。有两个选择：一是用传统CGI（经典仿真）——每一帧都能精确控制，但看起来不够真实；二是用AI视频生成（神经仿真）——画面非常真实，但无法精确控制角色的动作。ComSim的做法是：先用CGI（经典仿真）生成精确的动作画面，然后训练一个AI模型，学会把CGI画面"翻译"成真实画面，同时确保角色的动作保持不变。这样既获得了精确的动作控制，又得到了真实的视觉效果。

**Real-Sim-Real（真实-仿真-真实）数据增强流水线**：
1. **Real→Sim**：用少量真实数据在经典仿真中复现相同的场景和动作，生成配对的"真实视频-仿真视频"数据
2. **Sim→Real（训练神经仿真器）**：利用配对数据训练一个神经仿真器，学会将仿真视频转换为真实风格的视频
3. **Sim→Real（大规模生成）**：在经典仿真中生成大量多样化的动作-视频数据，然后用训练好的神经仿真器将它们全部转换为真实风格的"伪真实数据"

### 方法逐步拆解

**第一步：Real2Sim 数据采集（对齐）**

要训练一个能将仿真视频转换为真实视频的模型，首先需要配对的"（仿真视频，真实视频，动作序列）"数据。构建这样的数据集需要在三个方面进行精确对齐：
1. **背景和物体对齐**：将仿真中的桌面颜色、背景颜色调整到与真实采集平台一致；对物体使用数字孪生方法确保视觉一致性和真实尺度匹配。
2. **相机标定对齐**：确保仿真中的相机参数和位姿与真实世界完全一致。
3. **物体位置对齐**：在任务初始化时，定位真实场景中的物体位置并严格传递到仿真中。

通过以上三步，获得 10 个任务共 200 组配对数据用于训练。

**第二步：组合动态视频生成（训练神经仿真器）**

神经仿真器的核心是一个 DiT（Diffusion Transformer），它接受两个条件的引导：
- **控制动态（Control Dynamics）**：机器人的动作序列
- **视觉动态（Visual Dynamics）**：仿真中的观测图像

训练目标是生成与真实世界分布一致的视频，同时保持动作对齐。损失函数为：

$$L_{sim2real} = L_{video}(f_N(V_{sim}, A, \theta), V_{real})$$

其中 $L_{video}$ 衡量生成视频与真实视频之间的差异。

在采样生成阶段，控制动态和视觉动态的得分（score）被组合起来指导生成过程，这种方法称为"动态引导"（Dynamic Guidance）。

**第三步：基于规则的仿真数据大规模生成**

使用 RoboTwin（基于 SAPIEN 的双臂操作仿真环境）来大规模生成多样化轨迹：
- 定义一套"动作基元"（action primitives），如抓取、推动、对齐、堆叠等
- 使用 GPT-5 根据任务描述将动作基元组合成可执行代码
- 系统变化环境条件、物体初始状态和机器人动作，生成覆盖广泛场景的轨迹

**第四步：真实世界部署**

将经典仿真中生成的大量 $(V_{sim}, A)$ 数据通过训练好的神经仿真器转换为 $(V_{real}, A)$ 伪真实数据，然后与少量真实数据混合进行策略训练。

### 关键公式/算法解读

**公式1：神经仿真映射**

$$N(D_{sim}) \approx D_{real}$$

神经仿真函数 $N$ 将仿真数据 $D_{sim}$ 映射到接近真实数据分布 $D_{real}$ 的分布。关键是映射后的动作 $a_i$ 与原始动作保持一致：$a_i \approx a'_j$。

**算法1：真实世界部署流水线**

输入：仿真数据 $(V_{sim}, A)$ 和真实数据 $(V_{real}, A)$
输出：训练好的机器人策略 $\pi_{robot}$

1. 初始化仿真数据集 $D_{sim}$ 和真实数据集 $D_{real}$
2. 初始化伪真实数据集 $P_{pseudo} = \{\}$
3. 对每个仿真样本 $(V_{sim}, A) \in D_{sim}$:
   - 使用神经仿真器 $f_N$ 转换为伪真实数据并加入 $P_{pseudo}$
4. 组合数据：$D_{combined} = \alpha \cdot D_{real} + (1-\alpha) \cdot P_{pseudo}$
5. 在组合数据上训练策略 $\pi_{robot}$
6. 返回训练好的策略

其中 $\alpha$ 是真实数据的混合比例。

### 实验设计分析

**视频生成质量评估**：使用 PSNR、SSIM、CLIP Score、LPIPS、FID、FVD 等6个指标，在8个不同任务上评估。完整方法（Ours-Full）在所有指标上达到最优，FID=147.90，FVD=15.765，远优于基线。

**消融实验**：
- Ours-CD（仅控制动态）：效果最差，说明视觉动态在生成中占主导地位
- Ours-VD（仅视觉动态）：贡献了大部分 sim-to-real 迁移能力，但在动作精度和物理一致性上仍有缺陷
- Ours-Full（两者结合）：不仅实现了照片级真实的视觉生成，还通过控制动态的运动引导实现了精确的动作复现

**真实世界策略训练**：在仅10个真实演示的条件下，比较6种数据混合方案：
- **10 Real**：仅10个真实演示 → 成功率很低
- **20 Real**：20个真实演示 → 有所提升但仍有限
- **10 Real + 200 Sim（经典仿真）**：传统仿真数据帮助有限，受视觉-物理差距限制
- **10 Real + 200 Pseudo Real（ComSim）**：伪真实数据带来显著提升，ID任务成功率从6-12/30提升到23-29/30
- **200 Pseudo Real（零样本）**：完全不使用真实数据也有不错的性能

**泛化能力**：
- **空间布局泛化**：将物体移到训练时未见过的区域，只有ComSim的伪真实数据带来了显著的泛化能力提升
- **新物体泛化**：更换不同颜色、品牌的瓶子（如可口可乐、雪碧），ComSim同样表现出色

## 问题与动机

机器人学习面临数据瓶颈：真实数据采集成本高、覆盖有限；经典仿真数据虽然有精确的动作配对但视觉不真实；神经仿真数据虽然视觉真实但动作控制不精确，存在"幻觉"。现有方法无法同时获得"精确的动作-视频对齐"和"真实的视觉表现"，ComSim旨在通过组合两者的优势来解决这一矛盾。

## 核心方法

ComSim提出"组合仿真"范式，核心是一个闭环的 Real-Sim-Real 数据增强流水线。首先通过精密的数字孪生对齐（背景、相机、物体位置），构建配对的仿真-真实视频数据集。然后训练一个基于DiT的神经仿真器，以控制动态（动作）和视觉动态（仿真观测）为条件，将仿真视频转换为真实风格视频。训练好的神经仿真器可以将经典仿真中大规模生成的多样化轨迹数据转换为"伪真实数据"，与少量真实数据混合用于策略训练。

## 实验结果

- **数据集**: RoboTwin仿真数据集（200-2000条）、10-20个真实演示
- **主要指标**: PSNR, SSIM, CLIP Score, LPIPS, FID, FVD（视频质量）；任务成功率（策略性能）
- **关键结果**: Ours-Full FID=147.90, FVD=15.765（视频生成）；10 Real + 200 Pseudo Real 在ID任务上成功率高达28-29/30，OOD任务上12-17/30，远超基线
- **对比方法**: 经典仿真数据直接训练、基线视频生成模型、零样本模型、仅控制动态/仅视觉动态消融变体

## 局限性

1. **场景限制**：当前实验主要集中在桌面操作任务，尚未扩展到更复杂的机器人形态（如移动操作）。
2. **依赖配对数据**：需要构建精确对齐的"仿真-真实"配对数据集来训练神经仿真器，这个过程本身也需要一定的人工努力。
3. **神经仿真器泛化**：训练好的神经仿真器在遇到与训练数据差异很大的场景时，生成质量可能会下降。
4. **未探索无配对数据**：未来可以考虑利用无配对数据来增强神经仿真器的能力和泛化性。

## 个人思考

ComSim 的"组合仿真"思路非常优雅——它不试图在单个方法中解决所有问题，而是巧妙地组合两个已有方法的优势。这种"分而治之"的思路在工程实践中往往比"一个模型搞定一切"更加实用和可靠。

特别值得注意的是它的"Real-Sim-Real"闭环设计。先用少量真实数据来完成仿真与真实之间的"对齐"（通过数字孪生），然后用对齐后的仿真环境大规模生成数据，再用神经仿真器将这些数据"翻译"回真实域。这种设计使得数据规模可以几乎无限扩展，而真实数据的需求保持在最低水平。

与 HyperSim 相比，两者都是 sim-to-real 框架，但侧重点不同：HyperSim 注重在仿真侧提高数据质量和覆盖率（通过高保真渲染和对抗训练），而 ComSim 更注重将仿真数据"翻译"到真实域（通过神经仿真器）。两者可以互补。

一个值得思考的方向是：神经仿真器的"幻觉"问题是否能通过更强大的基础模型（如视频生成模型的进步）得到根本解决？如果能，那么"经典仿真+神经翻译"这个路线可能会成为 sim-to-real 的主流范式。
