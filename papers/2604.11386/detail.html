ComSim: Building Scalable Real-World Robot Data Generation via Compositional Simulation

Yiran Qin, Jiahua Ma, Li Kang, Wenzhan Li, et al. 2026-04-13
arXiv:2604.11386
组合仿真 机器人数据 Real-Sim-Real 神经仿真 动作-视频生成

一句话总结

ComSim 提出"组合仿真"新范式，通过融合经典仿真（精确的动作-视频配对）和神经仿真（真实感渲染），构建闭环 "真实→仿真→真实"数据增强流水线，仅需少量真实数据就能生成大规模、多样化且动作一致的训练数据。

背景与前置知识

▼

经典仿真 vs 神经仿真

经典仿真（MuJoCo, Isaac Lab, ManiSkill）使用物理引擎模拟交互，优势是精确记录动作-视频配对，缺点是视觉不真实。神经仿真使用视频生成模型生成数据，优点是视觉真实，缺点是存在"幻觉"——生成的视频可能在物理上不一致，导致动作与视频对应关系被破坏。

领域鸿沟

经典仿真保证了动作精确性但视觉不真实；神经仿真保证了视觉真实性但动作控制不精确。两者各自擅长解决领域鸿沟的不同方面。

扩散模型与 DiT

扩散模型通过逐步加噪然后学习去噪来生成新样本。DiT（Diffusion Transformer）将 Transformer 架构应用于扩散模型。控制动态指机器人动作序列；视觉动态指摄像头观测图像序列。

核心思想详解

▼

类比：想象你要为电影场景制作特效。传统CGI（经典仿真）每一帧都能精确控制但不够真实；AI视频生成（神经仿真）画面真实但无法精确控制角色动作。ComSim的做法是：先用CGI生成精确的动作画面，然后训练AI模型把CGI画面"翻译"成真实画面，同时确保角色动作保持不变。

Real-Sim-Real 流水线：

  • Real→Sim：用少量真实数据在仿真中复现场景和动作，生成配对的"真实视频-仿真视频"数据
  • Sim→Real（训练）：利用配对数据训练神经仿真器，学会将仿真视频转换为真实风格视频
  • Sim→Real（生成）：在经典仿真中生成大量多样化数据，用神经仿真器全部转换为"伪真实数据"

方法逐步拆解

▼

第一步：Real2Sim 数据采集

通过背景/物体、相机标定、物体位置三方面精确对齐，构建配对的 (仿真视频, 真实视频, 动作序列) 数据集。获取10个任务共200组配对数据。

第二步：组合动态视频生成（训练神经仿真器）

基于 DiT 架构，接受控制动态（动作）和视觉动态（仿真观测）双重条件引导，生成与真实世界分布一致的视频。优化目标是 L_{sim2real} = L_{video}(f_N(V_sim, A, θ), V_real)。

第三步：大规模仿真数据生成

使用 RoboTwin 仿真环境和 GPT-5 + 动作基元（action primitives）自动生成多样化轨迹。

第四步：真实世界部署

将大量仿真数据通过神经仿真器转换为伪真实数据，与少量真实数据混合训练策略。

关键公式/算法解读

▼

神经仿真映射：N(D_sim) ≈ D_real。神经仿真函数 N 将仿真数据映射到接近真实数据分布，关键是映射后的动作与原始动作保持一致。

部署算法概要：

  1. 初始化 D_sim 和 D_real
  2. 对每个仿真样本：用神经仿真器转换为伪真实数据并加入 P_pseudo
  3. 组合数据：D_combined = α·D_real + (1-α)·P_pseudo
  4. 在组合数据上训练策略 π_robot

实验设计分析

▼

视频生成质量：使用 PSNR, SSIM, CLIP Score, LPIPS, FID, FVD 等6个指标，在8个任务上评估。Ours-Full 在所有指标上最优（FID=147.90, FVD=15.765）。

消融实验揭示：仅控制动态（Ours-CD）效果最差；仅视觉动态（Ours-VD）贡献了大部分迁移能力；两者结合（Ours-Full）实现最佳效果。

真实世界策略训练：在仅10个真实演示条件下，200条伪真实数据 + 10个真实数据的效果远超同类方案。ID任务成功率从6-12/30提升到23-29/30。

泛化能力：无论是空间布局泛化还是新物体泛化，只有 ComSim 的伪真实数据带来了显著提升。

局限性

▼
  • 当前集中在桌面操作任务，尚未扩展到更复杂机器人形态
  • 依赖精确对齐的"仿真-真实"配对数据集来训练神经仿真器
  • 神经仿真器在遇到与训练数据差异很大的场景时生成质量可能下降
  • 未探索无配对数据的利用方式

个人思考

ComSim 的"组合仿真"思路非常优雅——不试图在单个方法中解决所有问题，而是巧妙组合两个已有方法的优势。这种"分而治之"的思路在工程实践中往往比"一个模型搞定一切"更加实用。与 HyperSim 对比：HyperSim 在仿真侧提高数据质量（高保真渲染+对抗训练），ComSim 更注重将仿真数据"翻译"到真实域（通过神经仿真器），两者可以互补。未来"经典仿真+神经翻译"路线可能成为 sim-to-real 的主流范式。
