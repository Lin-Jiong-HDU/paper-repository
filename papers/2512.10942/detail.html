VL-JEPA: Joint Embedding Predictive Architecture for Vision-language

Delong Chen, Mustafa Shukor, Théo Moutakanni, Willy Chung, Jade Yu, Tejaswi Kasarla, Yejin Bang, Allen Boloourchi, Yann LeCun, Pascale Fung 2026-02-02
arXiv:2512.10942
视觉语言模型 联合嵌入预测架构 视频理解 嵌入空间 选择性解码

一句话总结

VL-JEPA 提出了一种基于联合嵌入预测架构的视觉语言模型，通过在连续嵌入空间而非离散 token 空间中进行预测，用更少的可训练参数实现了更强的多模态任务能力。

背景与前置知识

▼

自回归视觉语言模型 (VLM)：当前主流的视觉语言模型（如 LLaVA、InstructBLIP、Qwen-VL）采用"token 生成"范式——接收视觉输入和文本查询，自回归地逐 token 生成文本回复。这种方式虽然直观，但存在两个根本问题：(1) 训练成本高，模型需要同时学习任务相关语义和任务无关的表面语言特征（如措辞风格、同义改写）；(2) 实时推理延迟大，必须完成整个 token 序列生成后才能获取语义。

JEPA (Joint Embedding Predictive Architecture)：由 Yann LeCun 倡导的架构范式，核心思想是在抽象的嵌入空间中进行预测，而非在原始数据空间中重建。之前的工作包括 I-JEPA（图像）、V-JEPA（视频）等，但它们只针对单一模态。VL-JEPA 是首个面向通用视觉语言任务的 JEPA 模型。

CLIP 式对比学习：CLIP 通过 InfoNCE 损失将图像和文本对齐到共享嵌入空间，支持零样本分类和跨模态检索。但它无法生成文本，任务覆盖有限。

InfoNCE 损失：对比学习中的经典损失函数，包含两部分——表示对齐项（拉近正样本对）和均匀性正则项（推开负样本对），天然防止表示坍塌。

核心思想详解

▼

VL-JEPA 的核心洞见可以用一个类比来理解：传统 VLM 就像让学生逐字默写答案，而 VL-JEPA 则是让学生理解答案的含义即可。

想象你问"这张图里有什么动物？"——答案可能是"一只猫"、"这是一只猫"、"图中有猫"等等。这些在 token 空间中是完全不同的序列（几乎无重叠 token），但它们的语义是一致的。传统 VLM 必须学习所有这些变体，浪费了大量建模能力。VL-JEPA 通过 Y-Encoder 将这些语义等价的回答映射到嵌入空间中的相近点，将多峰的离散分布压缩为紧凑的单峰连续分布，大大简化了学习目标。

这种设计还带来了一个独特优势：选择性解码。因为预测是在嵌入空间中非自回归进行的，模型可以持续输出语义嵌入流，仅在检测到语义发生显著变化时才调用轻量级文本解码器。这对于智能眼镜、机器人等实时视频应用至关重要。

方法逐步拆解

▼

步骤 1：视觉编码 (X-Encoder)

  • 使用冻结的 V-JEPA V2-S（304M 参数），将视频/图像压缩为视觉 token 序列
  • 输入以 256² 分辨率均匀采样帧，图像则复制填充
  • 这一步不做训练，保留预训练视觉编码器的强大能力

步骤 2：条件预测 (Predictor)

  • 用 Llama-3.2-1B 的后 8 层 Transformer 初始化（490M 可训练参数）
  • 接收视觉 token 和文本查询 token，使用双向注意力（非因果 mask）
  • 通过平均池化和线性投影，输出 1536 维的预测嵌入

步骤 3：目标嵌入 (Y-Encoder)

  • 用 EmbeddingGemma-300M 初始化，最大上下文 512 token
  • 学习率为全局学习率的 0.05 倍（关键超参数），防止初期预测质量差导致嵌入空间不稳定
  • 线性投影头输出 1536 维嵌入，与 Predictor 共享嵌入空间

步骤 4：推理解码 (Y-Decoder)

  • 仅在推理时使用，将预测嵌入翻译为可读文本
  • 轻量级设计，不参与主训练循环
  • 支持选择性解码：仅当嵌入变化超过阈值时触发

步骤 5：训练目标

  • 使用双向 InfoNCE 损失，Predictor 和 Y-Encoder 联合训练、互相学习
  • 在同一个 batch 内，每个预测嵌入与对应目标嵌入拉近（对齐），与其他目标嵌入推远（均匀性）

关键公式/算法解读

▼

训练损失 (InfoNCE)：

L = -log(exp(sim(Ŝ_Y, S_Y) / τ) / Σ exp(sim(Ŝ_Y, S_Y') / τ))

  • Ŝ_Y：Predictor 输出的预测嵌入
  • S_Y：Y-Encoder 编码的目标嵌入
  • sim(·,·)：余弦相似度
  • τ：温度参数
  • 分母对所有 batch 内负样本求和
  • 双向应用：同时计算 Predictor→Y-Encoder 和 Y-Encoder→Predictor 方向

这个损失的本质是：让预测嵌入和真实目标嵌入在 1536 维空间中"靠得近"，同时让不同样本的嵌入"散得开"，避免所有嵌入坍塌到同一个点。

实验设计分析

▼

为什么选这些数据集？

论文的实验设计覆盖了视觉语言模型的四大任务类型，每个都有明确目的：

  1. 视频分类（8 个数据集）：测试嵌入空间的判别能力，这是 CLIP 的强项
  2. 文本到视频检索（8 个数据集）：测试跨模态对齐质量
  3. VQA（4 个数据集）：测试生成和理解能力，这是传统 VLM 的强项
  4. WorldPrediction-WM：测试世界模型理解能力，需要从初始/终态图像推断动作

关键对比实验

  • 与 CLIP、SigLIP2、Perception Encoder 对比分类/检索（证明 JEPA 范式优于纯对比学习）
  • 与 InstructBLIP、QwenVL 对比 VQA（证明小参数也能匹敌大模型）
  • 严格受控实验：相同视觉编码器 + 相同训练数据，唯一区别是 token 预测 vs 嵌入预测

结果说明什么？

  • 嵌入预测比 token 预测更高效：0.5B vs 1B 可训练参数，性能翻倍
  • 单一架构统一多任务：不需要为分类/检索/VQA 设计不同模型
  • 在世界模型理解上超越 GPT-4o、Claude-3.5、Gemini-2.0，说明语义嵌入空间确实学到了更深层的视觉-语言关联

局限性

▼
  1. 评估范围有限：论文明确指出，VL-JEPA 的目标不是替代 VLM 的所有能力。在复杂推理、工具使用和智能体行为等任务上，当前的 token 生成模型仍然更强
  2. 模型规模未充分探索：虽然结果显示了参数和数据规模扩展的潜力，但作者没有进行大规模扩展实验
  3. 生成质量：嵌入预测→解码的方式可能在需要精确措辞的任务上不如直接 token 生成灵活
  4. Y-Decoder 依赖：生成任务仍需 Y-Decoder，其质量直接影响输出文本质量

个人思考

VL-JEPA 的核心贡献不是在某个 benchmark 上刷 SOTA，而是提供了一种根本不同的视觉语言建模范式。将学习目标从 token 空间提升到语义嵌入空间，与 Yann LeCun 倡导的"抽象表示学习"理念一脉相承。选择性解码对于智能眼镜、机器人等实时应用具有实际价值。WorldPrediction-WM 上超越 GPT-4o 等大模型尤其令人印象深刻——说明在特定任务上，精心设计的小模型可以在语义嵌入空间中胜过通用大模型。
