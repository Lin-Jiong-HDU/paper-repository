# Rein3D: Reinforced 3D Indoor Scene Generation with Panoramic Video Diffusion Models

## 基本信息

- **标题**: Rein3D: Reinforced 3D Indoor Scene Generation with Panoramic Video Diffusion Models
- **作者**: Dehui Wang, Congsheng Xu, Rong Wei, Yue Shi, Shoufa Chen, Dingxiang Luo, Tianshuo Yang, Xiaokang Yang, Wei Sui, Yusen Qin, Rui Tang, Yao Mu（上海交通大学、群核科技、大疆、香港大学）

- **arXiv 链接**: <https://arxiv.org/abs/2604.10578>
- **领域/关键词**: 3D reconstruction, 3DGS, 全景视频扩散, 室内重建, 强化学习, 场景补全

## 一句话总结
>
> Rein3D 提出了一种"修复-精化"循环范式，通过径向探索策略渲染不完善的全景视频，利用全景视频扩散模型修复缺失的几何和纹理，再将修复后的高保真视频回传更新全局 3DGS 场，实现了从单张全景图出发的高质量、全局一致的 3D 室内场景重建。

## 论文精读

### 背景与前置知识

**3D 高斯泼溅（3D Gaussian Splatting, 3DGS）**：一种高效的 3D 场景表示方法。场景由大量 3D 高斯原语（Gaussian Primitives）表示，每个原语有位置（均值）、协方差矩阵（控制形状和大小）、不透明度和颜色参数。渲染时通过 alpha 混合从前到后合成像素颜色。

**全景图（Panorama）**：360° 全方位图像，通常使用等距柱状投影（Equirectangular Projection, ERP）表示。全景图覆盖整个球面视野，但存在极地区域畸变严重的问题。

**视频扩散模型**：将扩散模型从图像扩展到视频，在空间维度的基础增加时间维度，能够生成帧间一致连续的动态内容。视频到视频（V2V）扩散模型以低质量视频为条件，生成高质量、时间一致的输出。

**径向探索**：从场景中心点出发，沿多个均匀分布的方向向外移动相机，拍摄不同视角的图像/视频，以覆盖场景中从中心视角被遮挡的区域。

**"修复-精化"（Restore-and-Refine）范式**：先从一个粗粒度的初始场景开始，渲染不完善的视图；然后用生成模型修复这些视图中的缺陷（空洞、模糊、伪影）；最后用修复后的高质量视图更新优化 3D 场景表示。循环迭代直至场景质量满足要求。

### 核心思想详解

Rein3D 的核心可以类比为**"先搭框架再修补细节"的 3D 重建方法**。

想象你站在房间中央拍了一张全景照片（360° 无死角）。基于这张照片和深度信息，你可以快速生成一个粗糙的 3D 模型——但这就像用一块布把所有东西大致盖住，当你走近细看时，会发现很多地方是空心的、模糊的或有裂缝的。

Rein3D 的做法是：**从房间中心出发，沿着不同方向往外走，边走边拍——但这次相机拍的不是新照片，而是从初始粗模型渲染出的视图。** 由于粗模型有很多漏洞，这些渲染视图也是有缺陷的。然后，用一个专门训练的"全景视频修复器"将这些有缺陷的视图修复为高质量、像素完整的画面。最后，这些修复后的高质量画面被当作"地面真值"反馈回去，不断优化初始的 3D 模型。

这个循环的关键优势在于：
1. **全局一致性**：全景图的 360° 视野确保每次移动都能看到整个场景，不会出现局部方法常见的累积漂移。
2. **利用视频扩散的时间一致性**：视频扩散模型不仅修复每帧的内容，还能确保帧与帧之间的连续性和一致性。
3. **简单高效的轨迹**：由于全景图覆盖一切，相机只需要做最简单的直线运动就能覆盖所有区域，不需要复杂的路径规划。

### 方法逐步拆解

#### 步骤 1：初始场景构建

**输入**：文本提示（如"一个现代简约卧室"）或用户提供的全景图。
**过程**：
- 如果是文本输入：使用 DiT360 生成高分辨率全景图，再利用 DA2 深度估计模型预测对应的深度图。
- 将全景 RGB 图像和深度图通过反投影（unprojection）提升为 3D 高斯原语集。每个像素被反投影为一个 3D 高斯原语，位置由深度决定，初始化为完全透明、各向同性的球体。

**输出**：一个粗糙的初始 3DGS 场景表示。

#### 步骤 2：径向轨迹生成与渲染

**目标**：有效发现粗模型中的缺失区域。
**过程**：
- 使用全景深度计算安全导航地图（可移动区域的二值掩码）。
- 设计 τ 条从场景中心均匀向外辐射的轨迹，动态优化全局方向以最大化累积移动范围。
- 沿这些轨迹渲染缩放（zoom-in）的 RGB-Alpha 全景视频。由于粗模型中存在空洞和伪影，这些视频包含视觉瑕疵。

**输出**：多段有缺陷的全景视频（每段 τ 帧）。

#### 步骤 3：全景视频修复网络（核心创新）

**网络架构**：基于 Wan2.1-1.3B DiT 骨干网络，通过上下文适配器（Context Adapter）架构适配全景域。

**条件注入机制**：
- **前景/背景分离**：利用 Alpha 通道将渲染视频分解为前景（已有内容）和背景（缺失区域），分别编码后在通道维度拼接。
- **上下文锚点（Context Anchor）**：由于轨迹从场景原点开始，第一帧全景图是完全可见的（Alpha=1）。将第一帧作为全局参考锚点，在时间维度上拼接到视频特征前。这确保生成过程始终锚定在初始场景几何上，防止时间漂移和颜色不一致。

**球形优化**：
- **纬度感知采样（Latitude-Aware Sampling）**：等距柱状投影在极地区域拉伸严重。在噪声初始化时，根据纬度对噪声坐标进行拉伸，使噪声密度与球面上的视觉信号密度匹配。
- **纬度衰减损失（Latitude-Decay Loss）**：极地像素在物理球面上对应更小的面积。引入权重掩码降低极地对损失的贡献，聚焦于信息丰富的赤道区域。
  W(ϕ) = λ + (1 - λ)·cos(ϕ)，ϕ 为纬度角。

**输出**：修复后的高质量全景视频序列。

#### 步骤 4：视频超分辨率与 3D 场景精化

**过程**：
- 使用 FlashVSR 对修复后的视频进行超分辨率，恢复高频细节。
- 将高分辨率全景帧投影为透视视图，作为伪地面真值（Pseudo Ground Truth）。
- 以这些伪地面真值为监督，对 3DGS 进行精化优化（15000 步），采用稳健的致密化策略和抗锯齿技术。

**输出**：完整、高质量、全局一致的 3D 室内场景。

#### 数据集：PanoV2V-15K

为支持全景视频扩散模型的训练，论文构建了包含 15,050 个室内场景的大规模数据集。每个场景包含：从线性轨迹渲染的地面真值全景视频、以及从粗 3DGS 渲染的有缺陷的条件视频。覆盖多种房间类型。

### 关键公式/算法解读

**3DGS 渲染方程**：

```
c(x) = Σ c_k(x) · α_k(x) · Π (1 - α_i(x))
```

从前往后对排序后的高斯原语进行 alpha 混合，合成像素颜色。c_k 是第 k 个原语的 RGB 值，α_k 是该原语的透明度。

**视频扩散前向过程**：

```
q(z_t | z_0) = N(z_t; √α_t · z_0, (1-α_t)I)
```

对干净视频潜在表示 z_0 逐步加噪。

**上下文条件注入**：

```
Z_condition = [Z_anchor, Z_video]
```

将锚点帧特征与视频特征在时间维度拼接。Z_anchor = E(I_0) 是第一帧全景图的编码。

**纬度感知采样**：

```
x' = x_c + (x - x_c) · cos(ϕ)
```

根据纬度 ϕ 拉伸噪声坐标，极地区域压缩比例更大。

**纬度衰减损失权重**：

```
W(ϕ) = λ + (1 - λ) · cos(ϕ)
```

λ 是超参数，默认 0.1。纬度越高权重越低，赤道区域权重最大。

### 实验设计分析

**视频修复实验**：

在从 PanoV2V-15K 中随机选择的 50 个视频片段上评估。对比方法包括 ProPainter（视频修复 Transformer）和 VACE（视频生成编辑基础模型）。评估指标包括 PSNR、SSIM、FVD（视频质量+时间一致性）以及加权的球形版本 WS-PSNR/WS-SSIM。

**结果**：Rein3D 在所有指标上最优。PSNR 23.77（VACE FT 22.18），WS-PSNR 23.45（VACE FT 21.89），FVD 5.26（ProPainter 14.56）。WS 指标的显著提升说明球形优化对全景数据的有效性。

**文本到 3D 场景生成**：

对比方法包括 WorldGen、EmbodiedGen、DreamScene360。定性结果显示：WorldGen 无法恢复缺失区域（严重空洞）；EmbodiedGen 仅限于小范围探索（伪影严重）；DreamScene360 模糊且时间不一致。Rein3D 在全视角变化下保持全局一致、高保真。

**全景图像到 3D 场景重建**：

在 Structured3D 数据集上评估。使用 Q-Align（视觉质量）、CLIP Score（语义一致）、NIQE/BRISQUE（无参考质量指标）。评估视角包括原点及通过最远点采样（FPS）选择的 4 个最远探索点，确保全面覆盖。

**结果**：Rein3D(τ=8) Q-Align 3.0629（EmbodiedGen 2.4786），BRISQUE 45.89（EmbodiedGen 53.51）。定性结果显示 EmbodiedGen 在远距离视角有大面积漂浮伪影，DreamScene360 有严重几何扭曲，Rein3D 保持结构完整。

**消融实验**：
- **无上下文锚点**：FVD 从 5.26 飙升至 7.75，WS-PSNR 下降约 1 dB。证明锚点对维持全局一致性的关键作用。
- **纬度衰减损失 λ 的敏感性**：λ=1（无衰减）WS-PSNR 降至 22.97；λ=0（完全忽略极地）也影响整体保真度。λ=0.1 取得最佳平衡。

## 问题与动机

从单张全景图重建完整的 3D 室内场景是一个病态问题——单个输入无法提供完整 3D 空间的全面信息，大量看不见的区域需要推理。现有方法要么产生局部合理但全局不一致的结果（图像基础方法），要么计算复杂度高且操作繁琐（多视图合成方法）。视频扩散方法虽然强有力，但受限于有限的视场角，覆盖整个场景需要复杂的路径规划。

## 核心方法

Rein3D 提出"修复-精化"范式：（1）从文本或图像生成全景图+深度，初始化粗 3DGS 场景；（2）设计径向探索策略从中心向外渲染有缺陷的全景视频；（3）基于 Wan2.1 的全景视频扩散模型（含上下文锚点和球形优化）修复视频；（4）超分辨率后将高质量帧作为伪地面真值精化 3DGS。构建了 PanoV2V-15K 大规模配对全景视频数据集。

## 实验结果

- **数据集**: PanoV2V-15K（15,050 室内场景）+ Structured3D（评估）
- **主要指标**: PSNR, SSIM, FVD, WS-PSNR, WS-SSIM（视频修复）；Q-Align, CLIP, NIQE, BRISQUE（3D 场景质量）
- **关键结果**: 视频修复全面领先——PSNR 23.77（VACE FT 22.18），FVD 5.26（ProPainter 14.56）；3D 场景 Q-Align 3.06（EmbodiedGen 2.48）；消融验证上下文锚点和纬度衰减损失的必要性
- **对比方法**: 视频修复——ProPainter, VACE；文本到 3D——WorldGen, EmbodiedGen, DreamScene360；图像到 3D——EmbodiedGen, DreamScene360

## 局限性

1. **依赖合成数据**：PanoV2V-15K 基于合成室内环境，与真实世界场景存在域差距。
2. **计算资源需求高**：4× H200 GPU 训练，包括视频扩散模型和 3DGS 精化，计算成本较高。
3. **线性轨迹限制**：当前使用直线轨迹，在特别复杂的空间布局中可能无法完全覆盖所有被遮挡区域。
4. **单帧全景锚点依赖**：如果初始全景图质量不佳（如生成瑕疵或深度估计错误），会影响整个流程。
5. **未支持语义编辑和动态场景**：目前限于静态场景重建，不支持物体编辑或动态变化。
6. **轨迹数量 τ 的选择**：τ=4 和 τ=8 之间性能差异不大，说明增加轨迹数量边际收益递减，但最优 τ 值可能需要根据场景调整。

## 个人思考

Rein3D 的"修复-精化"循环思路非常巧妙——它不试图直接从稀疏输入一次性生成完美 3D 场景，而是通过"渲染→修复→回传"的闭环逐步改进。这种渐进式的策略比端到端方法更稳健，也更容易融入现有 3DGS 管线。

上下文锚点（Context Anchor）是一个简单但极其有效的设计。将初始全景帧作为时间锚点，确保视频扩散过程始终锚定在初始几何上，防止长序列生成中的漂移。这个思路可以推广到其他视频生成任务中。

纬度优化的两个技术（纬度感知采样和纬度衰减损失）反映了论文对全景几何特性的深入理解。等距柱状投影的极地畸变是全景处理中的经典问题，但直接在扩散模型中处理这个问题的研究不多。这种"数据自适应"的噪声初始化方法比简单的数据增强更优雅。

一个潜在的有趣方向是扩展到动态场景——如果视频扩散模型能够"想象"场景中物体的运动（如移动椅子、开门），那么"修复-精化"循环不仅可以补全静态几何，还可以为具身智能体提供动态训练环境。
